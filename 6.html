Module 01 · 课程导论与项目总览
YOLO系列发展脉络
YOLO（You Only Look Once）系列是目标检测领域最具影响力的算法家族之一，经历了多个重要版本的演进：
2016
YOLOv1
开创性将检测视为回归问题，单次前向完成定位与分类
2017
YOLOv2
引入Anchor Box与Batch Normalization，提升召回率
2018
YOLOv3
多尺度预测 + Darknet-53骨干网络，成为工业界标准
2020
YOLOv4/v5
CSPDarknet + Mosaic增强，速度和精度达到新高度
2021+
YOLOv5-Lite
轻量化改进，适配移动端与边缘设备部署
技术背景
YOLOv5-Lite 基于 Ultralytics YOLOv5 进行轻量化改造，核心改进包括：使用 ShuffleNet 或 MobileNet 等轻量骨干网络替换 CSPDarknet，减少参数量和计算量，同时保持可接受的检测精度。适用于 Jetson Nano、树莓派、手机等资源受限平台。

Module 01 · 课程导论与项目总览
YOLOv5-Lite 技术定位与优势
YOLOv5-Lite 由社区开发者 ppogg 维护，旨在提供一种可在边缘设备上高效运行的目标检测方案。其核心设计理念是在模型大小、推理速度和检测精度之间取得平衡。
技术特征
  • 轻量骨干网络（ShuffleNetv2 / MobileNetv3 等）
  • FPN + PAN 特征金字塔结构
  • 解耦检测头（Decoupled Head）
  • 支持 TensorRT / NCNN / ONNX 导出
  • 模型体积可压缩至数 MB
适用场景
  • 嵌入式设备实时检测（Jetson Nano / TX2）
  • 移动端 App 集成（Android / iOS）
  • 边缘计算节点部署
  • 低延迟要求的工业检测
  • 无人机 / 机器人视觉感知
参考仓库
主仓库：https://github.com/ppogg/YOLOv5-Lite —— 建议课堂使用固定版本或教师提供的压缩包，避免因网络和版本变化导致不一致。

预训练权重文件下载
权重文件（.pt 格式）存储了模型在大型数据集（如 COCO）上预训练的参数。推理前必须下载对应版本的权重文件。
权重获取方式
# 方式一：从 Release 页面下载 # 访问 https://github.com/ppogg/YOLOv5-Lite/releases # 下载 yolov5-lite.pt 或对应变体权重 # 方式二：使用 wget 直接下载 wget https://github.com/ppogg/YOLOv5-Lite/releases/download/v1.0/yolov5-lite.pt -P weights/ # 方式三：运行 detect.py 自动下载（首次推理时触发） python detect.py --weights weights/yolov5-lite.pt --source data/images
权重文件组织规范
建议将所有权重文件统一存放于项目根目录下的 weights/ 文件夹中。目录结构为 YOLOv5-Lite/weights/yolov5-lite.pt。若该目录不存在，需手动创建。

Module 03 · 目录结构解析
顶层目录总览
YOLOv5-Lite 的项目根目录按功能划分为若干子目录和入口脚本。以下为典型结构（不同版本可能略有差异）：
YOLOv5-Lite/ ├── models/ # 网络结构定义、模型配置文件（.yaml） ├── data/ # 数据集配置、类别名称文件 ├── utils/ # 通用工具：数据处理、NMS、可视化、日志 ├── weights/ # 预训练权重或训练后权重（.pt 文件） ├── runs/ # 推理/训练输出目录（自动生成） ├── detect.py # 推理入口脚本 ├── train.py # 训练入口脚本 ├── test.py / val.py # 验证/测试入口脚本 ├── requirements.txt # Python 依赖清单 └── README.md # 项目说明文档
阅读策略
初次接触时，按 detect.py → utils/ → models/ → data/ 的顺序阅读。detect.py 是推理入口，沿着其函数调用链可以自然理解其他模块的作用。

Module 03 · 目录结构解析
models/ 目录详解
models/ 目录是模型结构的核心，包含网络架构定义和配置文件。这是整个项目中最重要的目录。
models/ ├── common.py # 通用模块：Conv、DWConv、SPP、Focus、C3 等基础组件 ├── yolo.py # 检测模型主干：Model 类、parse_model 函数 ├── export.py # 模型导出：ONNX、TorchScript、TensorRT └── yolov5l.yaml # 模型配置文件：深度、宽度、骨干网络结构参数
common.py 核心组件
  • Conv：标准卷积 + BN + SiLU
  • DWConv：深度可分离卷积（轻量关键）
  • C3：CSP 瓶颈模块
  • SPP：空间金字塔池化
  • Focus：下采样切片操作
yolo.py 核心功能
  • Model 类：整体模型容器
  • parse_model：解析 yaml 配置构建网络
  • Detect 类：检测头（输出层）
  • 前向传播逻辑编排

模型配置文件解析
YOLOv5-Lite 使用 YAML 格式的配置文件定义网络结构。以 models/yolov5l.yaml 为例，其核心参数如下：
# YOLOv5-Lite 模型配置示例 # parameters nc: 80 # 类别数量（COCO 数据集为 80 类） depth_multiple: 0.67 # 模型深度缩放系数 width_multiple: 0.75 # 模型宽度（通道数）缩放系数 # anchors anchors: - [10,13, 16,30, 33,23] # P3/8 特征图锚点 - [30,61, 62,45, 59,119] # P4/16 特征图锚点 - [116,90, 156,198, 373,326] # P5/32 特征图锚点 # backbone 骨干网络定义 backbone: [ [-1, 1, Focus, [64, 3]], # 0-P1 [-1, 1, Conv, [128, 3, 2]], # 1-P2 [-1, 3, C3, [128]], # 2 [-1, 1, Conv, [256, 3, 2]], # 3-P3 [-1, 6, C3, [256]], # 4 [-1, 1, Conv, [512, 3, 2]], # 5-P4 [-1, 9, C3, [512]], # 6 [-1, 1, Conv, [1024, 3, 2]], # 7-P5 [-1, 3, C3, [1024]], # 8 [-1, 1, SPPF, [1024, 5]], # 9 ]
配置参数解读
每行格式为 [from, number, module, args]：from 表示输入来自哪一层（-1 为上一层），number 为该模块重复次数（乘以 depth_multiple），module 为模块名称，args 为模块参数。

Module 04 · 推理命令与参数配置
命令行参数详解
detect.py 支持丰富的命令行参数，以下为最常用的参数及其说明：
参数 默认值 说明
--weights yolov5s.pt 模型权重文件路径，支持 .pt / .onnx / .engine 格式
--source data/images 输入来源：图片目录/单张图片/视频文件/摄像头ID/URL
--img 640 模型输入尺寸（像素），需为 32 的倍数
--conf 0.25 置信度阈值，低于此值的检测框将被过滤
--iou 0.45 NMS 的 IoU 阈值，控制重叠框的去除力度
--device '' 计算设备：'cpu' / '0'（GPU 设备号） / '0,1'（多 GPU）

Module 05 · 推理内部流程
Letterbox 预处理原理
YOLO 系列要求输入图像尺寸能被网络步长整除（通常为 32 的倍数，如 640×640）。Letterbox 操作在保持原始宽高比的前提下，对图像进行缩放并用边缘填充补齐到目标尺寸。
Letterbox 计算过程
def letterbox(img, new_shape=640, color=(114,114,114)): shape = img.shape[:2] # 原图尺寸 (h, w) r = min(new_shape[0] / shape[0], new_shape[1] / shape[1]) # 等比例缩放 new_unpad = (int(round(shape[1] * r)), int(round(shape[0] * r))) img = cv2.resize(img, new_unpad, interpolation=cv2.INTER_LINEAR) # 计算填充 dw = (new_shape[1] - new_unpad[0]) / 2 # 宽度填充 dh = (new_shape[0] - new_unpad[1]) / 2 # 高度填充 # 添加灰边填充 top, bottom = int(round(dh - 0.1)), int(round(dh + 0.1)) left, right = int(round(dw - 0.1)), int(round(dw + 0.1)) img = cv2.copyMakeBorder(img, top, bottom, left, right, cv2.BORDER_CONSTANT, value=color) return img, (r, r), (dw, dh)
坐标映射关键点
模型输出的检测框坐标基于 Letterbox 后的图像尺寸。后处理时必须将坐标从"模型输入尺寸"映射回"原图尺寸"。映射公式为：x_original = (x_model - dw) / r。这一步出错会导致框位置偏移或大小异常。

归一化与维度转换
OpenCV 读取的图像格式与 PyTorch 模型期望的输入格式存在差异，需要进行以下转换：
转换 1
BGR → RGB
OpenCV 默认 BGR 通道顺序，模型训练使用 RGB
转换 2
HWC → CHW
NumPy H×W×C → PyTorch C×H×W
转换 3
uint8 → float32
0-255 整数 → 浮点数
转换 4
0-255 → 0-1
除以 255 归一化到 [0,1] 区间
转换 5
添加 Batch 维
(3,H,W) → (1,3,H,W)
# 完整预处理代码（detect.py 中的实现） img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB) # BGR → RGB img = img.transpose(2, 0, 1) # HWC → CHW img = np.ascontiguousarray(img) # 内存连续化 img = torch.from_numpy(img).float() # uint8 → float32 img /= 255.0 # 归一化到 [0,1] img = img.unsqueeze(0) # 添加 batch 维度

  • 按置信度阈值过滤低分框
  • 按类别分组
