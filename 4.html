传统视觉方法：人工设计特征的局限
在深度学习普及之前，计算机视觉系统依赖人工设计的特征提取器。研究人员针对不同任务手工设计特征描述子：
特征类型典型描述子检测目标
边缘Canny、Sobel梯度变化剧烈的像素位置
角点Harris、FAST多个方向梯度均大的像素
纹理LBP、Gabor 滤波局部像素的统计模式
颜色颜色直方图全局或局部色彩分布
形状SIFT、HOG梯度方向分布的局部统计
当场景简单、光照稳定、目标形态固定时，这类方法高效且具备可解释性。然而在实际应用场景中——目标被遮挡、姿态与尺度变化、复杂背景干扰——人工规则很快变得脆弱，泛化能力严重受限。
核心矛盾：每一类新的检测场景都需要重新设计特征，特征的可迁移性极低，系统维护成本随场景复杂度线性增长。

深度学习的范式转变
深度学习的核心变化在于：特征不再由人工设计，而是由数据驱动学习。网络自动从原始像素中提取从低级到高级的层次化表示：
Layer 1–2
边缘、纹理、颜色斑块
Layer 3–5
局部形状、部件结构
Layer 6+
语义概念、完整对象
这种层次化学习模式使得模型能够: (1) 自动适应数据分布，减少人工调参；(2) 通过增加层数和通道数提升容量；(3) 预训练特征可迁移到相似任务。
Key Insight
浅层网络学习通用特征（边缘、纹理），深层网络学习任务特定特征（类别、语义）。这一特性使得预训练 + 微调的迁移学习范式成为可能——在 ImageNet 上预训练的模型可在少量标注数据上快速适配新任务。

神经元与线性变换
最基本的人工神经元可表示为线性变换加非线性激活：
y = f(W · x + b)
其中各符号含义：
符号维度含义
xd × 1输入特征向量（如图像的展平像素值）
Wm × d权重矩阵，连接输入到输出的线性映射
bm × 1偏置向量，提供平移自由度
f(·)—激活函数，引入非线性
重要性质：如果没有激活函数 f(·)，多层线性网络的堆叠仍然等价于一个线性变换。例如两层叠加 W₂(W₁x + b₁) + b₂ = (W₂W₁)x + (W₂b₁ + b₂)，本质上仍为 Wx + b 的形式。非线性激活是网络表达能力的根本来源。
神经元计算示意图
x₁ ──╮
x₂ ──╼ [ Σ(wᵢ·xᵢ) + b ] ──→ [ f(·) ] ──→ y
x₃ ──╯
 输入向量 x 经权重 w 加权求和后加偏置 b，再经非线性激活函数 f 得到输出 y

激活函数：Sigmoid 与 ReLU
Sigmoid
σ(x) = 1 / (1 + e−x)
输出范围 (0,1)，适用于概率输出层。早期神经网络的标准选择。
  • 优点：输出平滑可导，有概率解释
  • 缺点：两端饱和，梯度消失严重
  • 缺点：输出非零中心，影响梯度更新方向
ReLU
ReLU(x) = max(0, x)
正区间梯度恒为 1，是 CNN 中最常用的激活函数。
  • 优点：计算极简，缓解梯度消失
  • 优点：稀疏激活，约 50% 神经元输出 0
  • 缺点：负区间梯度为 0，神经元可能永久失活
梯度消失问题：Sigmoid 在 |x| > 4 时导数值接近 0，深层网络中梯度逐层衰减，导致浅层参数几乎无法更新。ReLU 的正区间梯度固定为 1，有效缓解了这一现象。
激活函数梯度特性对比
Sigmoid: x→−∞ 时 σ(x)→0, σ'(x)→0  |  x→+∞ 时 σ(x)→1, σ'(x)→0  →  两端饱和
ReLU: x<0 时 ReLU(x)=0, ReLU'(x)=0  |  x≥0 时 ReLU(x)=x, ReLU'(x)=1  →  单侧抑制
SiLU: x→−∞ 时 SiLU(x)→0, SiLU'(x)→0  |  x→+∞ 时 SiLU(x)→x, SiLU'(x)→1  →  平滑非单调

激活函数：SiLU/Swish 与综合对比
SiLU / Swish
SiLU(x) = x · σ(x)
SiLU = Sigmoid Linear Unit。Google 提出后广泛被 YOLOv5+、EfficientNet 等模型采用。
  • 平滑非单调：负区间有小幅负值输出
  • 自门控机制：输入 x 自身调控输出幅度
  • 在深层网络中效果优于 ReLU
  • 计算量略高于 ReLU（需计算 Sigmoid）
三种激活函数对比
函数输出范围梯度消失计算量
Sigmoid(0,1)严重中等（exp）
ReLU[0,∞)无（正半轴）极低
SiLU(−0.28,∞)轻微中等

损失函数：量化预测与真实的差距
均方误差（MSE）
MSE = (1/N) · Σ(yi − ŷi)²
适用于回归任务（边界框坐标、关键点定位）。对大误差惩罚较大，对离群值敏感。 
# PyTorch
loss_fn = nn.MSELoss()
loss = loss_fn(pred, target)
交叉熵（Cross-Entropy）
CE = −Σ yi · log(ŷi)
适用于分类任务。通常与 Softmax 配合使用，Softmax 将 logits 转化为概率分布。 
# PyTorch（自带 Softmax）
loss_fn = nn.CrossEntropyLoss()
loss = loss_fn(logits, labels)
选择依据：分类任务默认使用交叉熵损失——其梯度形式为 (ŷ − y)，当预测严重偏离真实值时梯度的幅度大，有利于快速修正；MSE 配合 Sigmoid 输出时梯度饱和严重，训练效率较低。边界框回归任务则通常使用 Smooth L1 Loss（Huber Loss）以降低离群值的梯度幅度。

实操：PyTorch 实现单神经元
# 定义环境与数据集
import torch
import torch.nn as nn
import torch.optim as optim

# 简单的线性神经元：y = Wx + b
class LinearNeuron(nn.Module):
    def __init__(self, input_dim, output_dim):
        super().__init__()
        self.fc = nn.Linear(input_dim, output_dim)
        self.activation = nn.Sigmoid()

    def forward(self, x):
        return self.activation(self.fc(x))

# 实例化模型与优化器
model    = LinearNeuron(input_dim=10, output_dim=1)
optimizer = optim.SGD(model.parameters(), lr=0.01)
loss_fn   = nn.MSELoss()

# 模拟数据
X = torch.randn(32, 10)
y = torch.randn(32, 1)

# 一次训练迭代
optimizer.zero_grad()          # 清空梯度
pred = model(X)                # 前向传播
loss = loss_fn(pred, y)        # 计算损失
loss.backward()                # 反向传播
optimizer.step()               # 更新参数

print(f"Loss: {loss.item():.4f}")
训练指令：将代码保存为 neuron.py，在终端执行 python neuron.py。需 PyTorch 2.0+，安装命令 pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118（CUDA 11.8 版）或 pip install torch torchvision（CPU/MPS 版）。

卷积操作：局部连接与参数共享
三个关键特性
  • 局部连接：每个神经元只连接输入的局部邻域（卷积核大小 3×3 或 5×5），而非全连接的全部像素
  • 参数共享：同一卷积核在整个输入图上滑动，参数量与输入尺寸无关
  • 平移等变性：输入特征平移 k 像素，输出特征图相应位置也平移 k 像素
S(i,j) = (I ∗ K)(i,j) = ΣmΣn I(i+m,j+n) · K(m,n) 
# PyTorch 2D 卷积层
import torch.nn as nn

# 参数：(in_channels, out_channels, kernel_size)
conv = nn.Conv2d(
    in_channels=3,       # RGB 输入
    out_channels=32,     # 输出通道数
    kernel_size=3,       # 3×3 卷积核
    stride=1,            # 滑动步长
    padding=1,           # 外围补零
    bias=True
)

# 输入: (batch, channels, height, width)
x = torch.randn(4, 3, 224, 224)
out = conv(x)            # → (4, 32, 224, 224)

# 参数量计算: 3×32×3×3 + 32 = 896
total_params = sum(p.numel() for p in conv.parameters())
print(f"Params: {total_params}")
如果使用全连接层处理 224×224×3 图像：假设隐藏层 1024 个神经元，参数量 = 224×224×3×1024 ≈ 1.54 亿，远超过卷积的 896 个。卷积通过局部连接 + 参数共享实现了高效的参数利用。
3×3 卷积核滑动示意
输入特征图 5×5  →  3×3 卷积核  →  输出特征图 3×3
每个输出位置对应输入上 3×3 邻域的加权求和，步长 stride=1 时输出尺寸 = (5−3)/1 + 1 = 3

通道与特征图
输入图像通常包含 RGB（3 通道）或 BGR 格式。卷积层输出通道数由本层卷积核数量决定——每使用一个卷积核就生成一个通道的特征图。
Feature Map
每个通道可理解为一种特征响应图——输入在特定特征模式上的激活强度。浅层通道响应边缘、纹理和颜色斑块；深层通道响应局部形状乃至语义部件。
从第 1 层到第 L 层，通道数通常逐渐增加（例如 64 → 128 → 256 → 512），同时特征图空间分辨率逐渐降低（224 → 112 → 56 → 28 → 14 → 7）。这种由宽到深的结构设计兼顾了计算效率与表示能力。
典型 CNN 通道演化
层组输入通道输出通道分辨率
Conv1364112×112
Conv26412856×56
Conv312825628×28
Conv425651214×14
Conv55125127×7
分组卷积与深度可分离卷积：标准卷积的计算量 = K² × Cin × Cout × H × W。深度可分离卷积将此拆分为 depthwise（K² × Cin × H × W）+ pointwise（Cin × Cout × H × W），参数量降低约 1/Cout + 1/K²。MobileNet、EfficientNet-Lite 等轻量模型采用此设计。

感受野（Receptive Field）
感受野定义为特征图上一个点对应到输入图像上的区域大小。网络越深，感受野越大，神经元能看到的上下文范围越广。
rout = rin + (k − 1) · Πi=1n−1 si
其中 rin 为输入感受野，rout 为输出感受野，k 为卷积核大小，si 为第 i 层的步长乘积。
层号操作步长感受野输出分辨率
1Conv 3×313×3224×224
2Conv 3×315×5224×224
3MaxPool 2×226×6112×112
4Conv 3×3110×10112×112
目标检测中的意义：检测算法需要在不同层级的特征图上检测不同尺度的目标。大目标需要大感受野捕获完整上下文，小目标则在浅层高分辨率特征图上检测。FPN（特征金字塔网络）的核心贡献即在于构建含多尺度感受野的特征层次。
感受野叠加过程（3×3 Conv × 3 层）
Layer 1: 3×3  →  Layer 2: 5×5  →  Layer 3: 7×7
每增加一层 3×3 卷积（stride=1），感受野扩大 2 个像素。三层堆叠后中心神经元可"看到"输入上 7×7 的区域。

池化与步长卷积
池化（Pooling）
池化层在空间窗口内进行下采样：
类型操作特点
最大池化取窗口内最大值保留最强响应，平移鲁棒
平均池化取窗口内平均值保留背景信息，平滑过渡
池化的三个作用：(1) 降低分辨率减少计算量；(2) 扩大下一层感受野；(3) 提供局部平移不变性。
步长卷积
使用 stride=2 的卷积替代池化层，是现代检测网络的常见做法：
  • strided conv：可学习下采样（vs 池化的固定规则）
  • 支持残差连接的直通路径
  • 在 YOLO、ResNet 中完全替代池化
# 最大池化
maxpool = nn.MaxPool2d(
    kernel_size=2, stride=2)

# 步长=2 的卷积（替代池化）
strided_conv = nn.Conv2d(
    in_channels=128,
    out_channels=256,
    kernel_size=3, stride=2,
    padding=1)

前向传播（Forward Pass）
输入图像依次经过各网络层：卷积层提取特征 → 激活函数引入非线性 → 池化/步长卷积降低分辨率 → 全连接层映射到输出空间 → 输出层产生预测类别或边界框。

x = input_image                  # shape: (B, 3, 640, 640)

x = conv3(x);  x = relu(x)      # → (B, 256, 80, 80)
x = flatten(x)                   # → (B, 256*80*80)
output = fc(x)                   # → (B, num_classes)

反向传播与链式法则
反向传播计算损失函数对各层参数的梯度。依据微分的链式法则，将输出层的误差逐层反向传播至各层：
∂L/∂W(l) = ∂L/∂y(L) · ( Πk=l+1L ∂y(k)/∂y(k−1) ) · ∂y(l)/∂W(l)
图中用计算图（computational graph）实现——每个操作节点保存前向结果，反向时利用链式法则沿图回溯。PyTorch 的 autograd 引擎自动完成这一过程：
计算图示意：y = x² + 3x
x ──→ [²] ──→ ① ──→ [+] ──→ y
x ──→ [×3] ──→ ② ──→ ↑
反向：∂y/∂x = 2x + 3，x=2 时梯度为 7 
# PyTorch 自动求导示例
x = torch.tensor([2.0],
    requires_grad=True)
y = x ** 2 + 3 * x          # forward
y.backward()                # backward
print(x.grad)               # dy/dx = 2x+3 = 7.0

# 训练中清空梯度
optimizer.zero_grad()       # 重要！否则梯度累积
loss.backward()             # 填充 .grad
optimizer.step()            # 更新参数
梯度消失 vs 梯度爆炸：如果链式乘积中的多项小于 1，深层网络的梯度会指数级收缩（梯度消失，Sigmoid 的典型问题）；如果多项大于 1，梯度指数级增长（梯度爆炸）。Batch Normalization 和残差连接（ResNet 的 skip connection）是缓解二者的核心设计。

过拟合：成因与诊断
过拟合（Overfitting）指模型在训练集上表现良好但在验证/测试集上效果显著下降。表现为训练 loss 持续下降而验证 loss 停滞或上升。
常见成因
  • 数据量太少：样本不足以覆盖真实分布
  • 标注不一致：噪声标注使模型学到错误关联
  • 场景过于单一：训练集缺乏多样性
  • 模型容量过大：参数量远超数据量需求
  • 训练轮数过多：在训练集噪声上过度拟合
诊断方法
  • 对比训练 loss 与验证 loss 曲线
  • 观察验证集 mAP 是否先升后降
  • 检查训练集类别分布是否均衡
  • 随机抽样检查标注质量
  • 可视化预测结果——高置信度+错误预测可能是过拟合信号
典型模式：训练 loss 持续下降至近 0，但验证 loss 在第 N 个 epoch 后止跌回升，验证 mAP 也从峰值下降。此时模型开始记忆训练数据中的噪声模式，而非学习可泛化的特征。
典型过拟合曲线示意图
▓ 训练损失 持续下降  |  ▓ 验证损失 先降后升（过拟合分界点）
↳ 最佳早停点应在验证损失止跌回升之前，约第 5–7 轮

正则化与早停
权重正则化
L2 正则（权重衰减，Weight Decay）在损失函数中加入权重向量的 L2 范数：
L' = L + (λ/2) · Σ ||W||²
L1 正则则鼓励稀疏权重（部分权重变为 0），适合特征选择。L2 正则更常用，等价于在优化器中设置 weight_decay 参数。
Dropout
训练时以概率 p 随机丢弃神经元，使网络不依赖特定神经元通路，类似集成学习的 Bagging 效果。推理时所有神经元保留但乘以 (1−p)。

IoU：预测框与真实框的交并比
IoU（Intersection over Union）衡量定位精度：
IoU = Area(Bpred ∩ Bgt) / Area(Bpred ∪ Bgt)
IoU = 1 表示完全重合，IoU = 0 表示无重叠。常用阈值 IoU ≥ 0.5 判定检测正确。 
# IoU 计算
def compute_iou(box1, box2):
    # box: [x1, y1, x2, y2]
    xi1 = max(box1[0], box2[0])
    yi1 = max(box1[1], box2[1])
    xi2 = min(box1[2], box2[2])
    yi2 = min(box1[3], box2[3])

    inter = max(0, xi2-xi1) * \
            max(0, yi2-yi1)
    area1 = (box1[2]-box1[0]) * \
            (box1[3]-box1[1])
    area2 = (box2[2]-box2[0]) * \
            (box2[3]-box2[1])
    union = area1 + area2 - inter
    return inter / union
IoU 可视化
▓ 真实框   ▓ 预测框   ▓ 交集区域
IoU = 交集面积 / 并集面积 = 重叠程度
课堂计算练习
预测框 Bp = [10, 10, 50, 50]，真实框 Bgt = [20, 20, 60, 60]：
交集: xi1=20, yi1=20, xi2=50, yi2=50
交集面积: 30 × 30 = 900
Bp 面积: 40 × 40 = 1600
Bgt 面积: 40 × 40 = 1600
并集: 1600 + 1600 − 900 = 2300
IoU = 900 / 2300 ≈ 0.391
IoU 变体: GIoU（引入最小外接矩形惩罚）、DIoU（考虑中心点距离）、CIoU（综合中心点 + 宽高比）。YOLOv5+ 默认使用 CIoU Loss 训练回归分支。

Precision 与 Recall
基于 TP（真正例）、FP（假正例）、FN（假负例）定义：
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
Precision 高：模型预测出的检测框中大部分是正确目标（误检少）
Recall 高：大部分真实目标被检测到（漏检少）
混淆矩阵示例
预测为正预测为负
真实为正TP = 85FN = 15
真实为负FP = 10TN = —
Precision = 85 / (85+10) = 0.895
Recall = 85 / (85+15) = 0.850
场景权衡：工业安全检测更重视 Recall（漏检可能导致安全事故），但误检过多导致系统频繁报警不可用。一般在 Recall ≥ 0.9 的前提下尽量提升 Precision。

AP 与 mAP
AP（Average Precision）：单个类别在 Precision-Recall 曲线下的面积。通过在不同置信度阈值下计算 Precision、Recall，绘制 PR 曲线后求积分。
AP 计算（COCO 标准）
在 Recall 轴上的 101 个插值点（0, 0.01, 0.02, …, 1.00）取对应位置的最大 Precision，计算平均值。相比 VOC 的 11 点插值更精细。在实际实现中，检测结果按置信度降序排列，逐样本更新 TP/FP 从而生成 PR 曲线。
指标IoU 阈值意义
mAP@0.50.5宽松定位要求，常用于 PASCAL VOC
mAP@0.750.75严格定位要求
mAP@0.5:0.950.5~0.95，步长 0.05COCO 标准主指标，10 个阈值的平均
mAP@S / M / LmAP@0.5:0.95按目标面积分小/中/大分别统计
mAP@0.5:0.95 更严格：要求模型在较精确的定位下仍保持检测能力。一个小目标 IoU 从 0.5 提升到 0.75 需要定位精度大幅改善。如果模型 mAP@0.5 高但 mAP@0.75 低，说明定位精度不够，尤其在工业精确测量场景下不可用。

PR 曲线与 F1 Score
PR 曲线以 Recall 为横轴、Precision 为纵轴。曲线的右上方面积越大，模型性能越好。理想的 PR 曲线在 Recall 升高时 Precision 保持不变。
F1 = 2 · P · R / (P + R)
F1 Score 是 Precision 和 Recall 的调和平均，在二者均衡时取最大值。对比算术平均，调和平均对低值更敏感——若 P=0.9、R=0.5，F1=0.64，而算术平均为 0.7。

▓ 曲线下面积 = AP  |  理想曲线：Recall 升高时 Precision 保持高位
↳ 高 AP 意味着模型在 Recall 从 0→1 的全程都保持高 Precision

# train_demo.py — 完整可运行的训练示例
import torch, torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

# 生成模拟数据
X = torch.randn(1000, 3, 32, 32)
y = torch.randint(0, 10, (1000,))

dataset = TensorDataset(X, y)
loader = DataLoader(dataset, batch_size=64, shuffle=True)

# 简单 CNN 模型
class DemoCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = nn.Sequential(
            nn.Conv2d(3, 16, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),        # → 16×16×16
            nn.Conv2d(16, 32, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),        # → 32×8×8
            nn.AdaptiveAvgPool2d(1),# → 32×1×1
            nn.Flatten(),
            nn.Linear(32, 10),
        )
    def forward(self, x):
        return self.conv(x)

model = DemoCNN()
opt = torch.optim.Adam(model.parameters(), lr=0.001)
loss_fn = nn.CrossEntropyLoss()

# 训练循环
model.train()
for epoch in range(10):
    total_loss = 0
    for xb, yb in loader:
        opt.zero_grad()
        pred = model(xb)
        loss = loss_fn(pred, yb)
        loss.backward()
        opt.step()
        total_loss += loss.item()
    avg_loss = total_loss / len(loader)
    print(f"Epoch {epoch+1}: loss={avg_loss:.4f}")
保存为 train_demo.py 后在终端执行：python train_demo.py。输出应为 10 行逐 epoch 的 loss 值且呈下降趋势。
