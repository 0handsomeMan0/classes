数据流详解

请求数据流：从前端到后端到模型

一次完整的检测请求经过 6 个步骤，每一步都有明确的模块边界和职责：

用户选择文件 + 调节阈值 —— 前端构造 FormData，包含 file 字段和 conf 字段。
前端发送 POST 请求 —— 使用 fetch() 异步发送至 /detect/image，设置 Content-Type: multipart/form-data。
FastAPI 接收并校验 —— UploadFile 解析文件内容，校验后缀名和文件大小，通过后写入上传目录。
调用推理函数 —— 将本地路径和置信度传入 detect_image()，此函数内部调用 YOLO 模型。
YOLO 执行推理 —— 加载权重、预处理图像、前向传播、非极大值抑制、绘制标注框并保存结果图。
返回 JSON 结果 —— 包含结果图路径、检测列表（类别+置信度+坐标）、推理耗时（毫秒）。

核心端点

检测接口实现：文件上传与参数接收

/detect/image 端点接收图片文件和置信度参数，执行校验后调用推理模块：

from fastapi import FastAPI, UploadFile, File, Form from pathlib import Path app = FastAPI() UPLOAD_DIR = Path("uploads") OUTPUT_DIR = Path("outputs") UPLOAD_DIR.mkdir(exist_ok=True) OUTPUT_DIR.mkdir(exist_ok=True) @app.post("/detect/image") async def detect_image_api( file: UploadFile = File(...), conf: float = Form(0.25), ): # 1. 校验文件后缀 suffix = Path(file.filename).suffix.lower() if suffix not in [".jpg", ".jpeg", ".png"]: return {"success": False, "message": "unsupported file type"} # 2. 保存上传文件 input_path = UPLOAD_DIR / file.filename input_path.write_bytes(await file.read()) # 3. 调用推理函数 result = detect_image(str(input_path), conf=conf) return result

注意：以上代码为教学骨架。生产环境需要补充文件名去重（UUID 或时间戳前缀）、文件大小限制、更严格的安全检查。

请求校验

输入校验的完整实现

健壮的 API 必须在入口处对请求进行全面校验，三行校验代码可避免绝大多数运行时错误：

import uuid from fastapi import HTTPException ALLOWED_SUFFIXES = {".jpg", ".jpeg", ".png"} MAX_FILE_SIZE = 16 * 1024 * 1024 # 16 MB @app.post("/detect/image") async def detect_image_api( file: UploadFile = File(...), conf: float = Form(0.25), ): # 校验 1: 文件后缀名白名单 suffix = Path(file.filename).suffix.lower() if suffix not in ALLOWED_SUFFIXES: raise HTTPException(400, "Unsupported file type. Allowed: JPG, PNG") # 校验 2: 文件大小上限 content = await file.read() if len(content) > MAX_FILE_SIZE: raise HTTPException(400, "File too large. Max: 16MB") # 校验 3: 置信度范围 conf = max(0.01, min(0.99, conf)) # 写入：使用 UUID 文件名避免冲突 unique_name = f"{uuid.uuid4().hex}{suffix}" input_path = UPLOAD_DIR / unique_name input_path.write_bytes(content) result = detect_image(str(input_path), conf=conf) return result

API 规范

统一的 JSON 响应格式设计

后端所有接口遵循统一的 success/data 或 success/message 格式，方便前端统一处理：

成功响应

{ "success": true, "data": { "result_image": "outputs/abc123_result.jpg", "elapsed_ms": 42, "detections": [ { "class_name": "helmet", "confidence": 0.91, "bbox": [120, 80, 240, 210] } ] } }

错误响应

{ "success": false, "message": "unsupported file type", "code": 400 }

响应字段说明：

success — 布尔值，前端以此判断结果状态。
data — 成功时携带的实际数据。
message — 错误时的可读描述。
code — 错误码（400/403/500），便于前端分类处理。

SECTION 04 · 模型推理函数封装

将推理逻辑与 API 层分离

不要在 API 函数中堆砌模型加载、预处理、推理、后处理等细节。抽象为独立函数后，API 层只需关心 HTTP 协议，推理层只需关心模型逻辑——职责分离，测试独立，替换灵活。

核心原则：API 不 import 任何深度学习库。API 调推理函数，推理函数调模型。这一层隔离使得在替换检测模型时，API 代码无需任何变更。

抽象层

推理抽象层：支持模型无缝替换

通过定义统一的推理接口，可实现不同检测模型的切换而不影响 API 层代码。两种常见的抽象方式：

方式一：函数式抽象

detect_image_v2() 替代 detect_image()，二者签名一致。
API 层只需修改 import 行即可切换。
适用于小规模项目、快速迭代。

方式二：类抽象（推荐）

定义 BaseDetector 抽象基类，所有模型实现必须实现 detect() 方法。
通过配置文件注册模型实例，运行时动态选择。
适用于支持多模型选择的中大型项目。

# 抽象基类方式（detector.py） from abc import ABC, abstractmethod class BaseDetector(ABC): @abstractmethod def detect(self, image: np.ndarray, conf: float) -> list: pass class YOLODetector(BaseDetector): def detect(self, image, conf): results = self.model(image, conf=conf) return self.parse_results(results)

数据持久化

检测记录管理：存储与查询

每次检测结果应持久化存储，支持历史回溯。最简单的方案是使用 JSON 文件存储记录列表：

import json, datetime from pathlib import Path RECORDS_FILE = Path("records") / "detections.jsonl" RECORDS_FILE.parent.mkdir(exist_ok=True) def save_record(filename: str, result: dict): record = { "timestamp": datetime.datetime.now().isoformat(), "filename": filename, "elapsed_ms": result["data"]["elapsed_ms"], "detections": result["data"]["detections"], } with open(RECORDS_FILE, "a") as f: f.write(json.dumps(record, ensure_ascii=False) + "\n") def get_history(limit: int = 20) -> list: records = [] if not RECORDS_FILE.exists(): return [] with open(RECORDS_FILE) as f: for line in f: line = line.strip() if line: records.append(json.loads(line)) return records[-limit:]

存储格式选择：JSON Lines（每行一条 JSON）便于追加写入和按行读取；SQLite 适用于更复杂的查询需求（按时间、类别、模型筛选）。
