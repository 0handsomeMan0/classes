关键概念

算法实验 · 工程交付 · 可复现

课程定位

从"跑通模型"到"交付系统"

多数教程止步于"用预训练模型跑通一张图片"。但工业级项目需要你回答的远不止这些：

❌ 学校/教程视角

下载预训练权重 → 对一张测试图片推理 → 输出结果 → 完成。

局限：数据来源、标注组织、推理速度与复现方式均未被纳入工程闭环。

✅ 工程交付视角

需求分析 → 数据采集/清洗 → 标注/校验 → 训练/调优 → 评估/测试 → 封装/部署 → 文档/演示。

核心观点。 机器视觉 ≠ 调用 detect() 函数。机器视觉 = 围绕业务需求形成可验证、可复现、可展示的系统。
这里强调一个认知升级：同学们可能看过很多"10 行代码实现目标检测"的教程，但那些只是 demo。企业要的不是你跑通一张猫的图片，而是你能否构建一个完整的工程流程——从数据到部署，每个环节都有交付物。这是本课程区别于网上教程的根本。

贯穿案例

工地安全帽检测平台

本学期我们将围绕一个真实工程问题展开：给定工地监控画面，自动检测工人是否佩戴安全帽。这不是一个跑通就结束的练习，而是一个完整的项目周期。

业务对象

人、安全帽、未佩戴安全帽的人员（违规）。三类检测目标。

输入数据

公开数据集（如 Safety-Helmet-Wearing-Dataset）、企业样例、自采摄像头画面。

模型输出

目标边界框坐标 (x,y,w,h)、类别、置信度分数、单帧检测耗时。

产品形态

上传图片/视频检测界面 或 本地摄像头实时检测演示。

这个案例会贯穿本学期所有模块。从今天搭建环境开始，到后面标注数据、训练 YOLO 模型、部署 web 界面，最后做一个完整的演示视频+答辩 PPT。让学生明白他们不是在学孤立的知识点，而是在建一个真正能用的东西。

工程思维训练

围绕安全帽检测的 5 个核心问题

一个 AI 项目不是"跑通一个 demo"，而是回答以下问题：

❶ 业务对象定义

人、安全帽、未佩戴安全帽的人员。需明确检测粒度、类别边界与业务告警目标。

❷ 数据来源设计

公开数据集、企业提供样例、自采摄像头画面。每种来源的优缺点和数量级要求。

❸ 模型输出规范

边界框 (x,y,w,h) + 类别 (person/hat/no-hat) + 置信度 + 检测时间。这 4 项信息缺一不可。

❹ 产品形态规划

上传图片检测 → 上传视频检测 → 实时摄像头检测。三个递进阶段。

❺ 交付材料清单

代码仓库、模型权重、数据说明文档、测试报告、演示视频、答辩 PPT。

这 5 个问题值得花 3-5 分钟讨论。让同学们先自己想答案，再揭晓。重点是让他们意识到：跑通模型只完成了 20% 的工作。数据、部署、文档、演示才是真正决定项目成败的部分。

行业场景 ①

工业质检 · 最成熟的落地领域

行业场景 1/5

工业质检 · AI 最早落地的领域

传统机器视觉依赖人工设计特征（边缘检测、模板匹配），对环境、光照、角度要求苛刻。深度学习带来了质的飞跃。

应用举例

手机屏幕划痕检测（分辨率要求 0.1mm）、PCB 焊点缺陷检测、纺织面料色差评级、铸件气孔/裂纹识别。

技术挑战

缺陷样本极少（正负样本比可达 10000:1）→ 少样本学习 / 数据增强。实时性要求：产线节拍 ≤ 200ms/件。

关键数据

据 MarketsandMarkets 报告，2025 年全球工业视觉市场规模预计达 159 亿美元，年复合增长率 8.5%。国内已有上百家创业公司切入该赛道。

思考题。 手机屏幕质检系统若需检测 0.1mm 划痕，应根据 6.7 英寸屏幕尺寸、视场范围与像素分辨率估算相机精度需求。
工业质检是视觉落地最成熟的领域，没有之一。可以讲一个具体案例：某手机代工厂原来 500 人质检团队，引入 AI 质检后缩减到 50 人，漏检率从 1.2% 降到 0.05%。但关键难点是缺陷样本太少——正常屏幕 10000 片才有一片有划痕，这导致训练数据极度不平衡。

行业场景 ②

安防巡检

行业场景 2/5

安防巡检 · 从"事后回看"到"实时预警"

安防是 CV 最大的单一市场。传统安防靠保安盯多块屏幕，研究表明 22 分钟后注意力下降 95%。AI 替代的是这件事。

应用场景

安全帽/反光衣检测（本节课贯穿案例）、烟火检测（周界防火）、区域入侵检测（电力/化工禁区）、人员倒地检测（养老院、地铁站）。

技术要点

多路视频流实时处理（4-32 路）、低延迟（<500ms）、夜视/红外场景适配、跨摄像头目标追踪（ReID）。

行业现状。 海康威视、大华股份等企业已实现基于AI的安防产品规模化。2024 年中国 AI 安防市场规模超过 800 亿元人民币，硬件+算法+平台一体化趋势明显。
安防巡检和我们贯穿案例直接相关。实际部署时需要考虑的远不止模型精度——多路摄像头怎么调度、夜视画面效果差怎么办、检测到了违规怎么推送到保安手机。这些都是工程问题。

行业场景 ③

零件分拣 · 机械臂+视觉

行业场景 3/5

零件分拣 · 机器人+视觉的黄金搭配

"眼睛"（视觉）告诉"手"（机械臂）"抓什么、在哪抓、怎么抓"。这是工业 4.0 最典型的场景之一。

典型流程

相机拍摄传送带 → 视觉算法识别零件类别和位姿 → 计算抓取点 (x,y,θ) → 机械臂执行抓取 → 放入对应料仓。整周期 < 2 秒。

技术栈

2D/3D 相机标定 → 目标检测/实例分割 → 位姿估计 (6-DoF) → 手眼标定 → 运动规划。涉及视觉+机器人两个领域的交叉。

课程链接。 本课程将覆盖检测和分割部分。位姿估计和手眼标定属于进阶内容，有兴趣的同学可以在期末项目中选方向深入研究。
零件分拣有个很好的 demo：用摄像头拍乐高积木，让机械臂按颜色分类。这个可以作为一个课后兴趣项目推荐给学生。技术关键点在于"手眼标定"——相机的坐标系和机械臂的坐标系怎么对齐。

行业场景 ④

智慧交通

行业场景 4/5

智慧交通 · CV 离消费者最近的场景

从车牌识别到自动驾驶，交通领域是普通消费者感受 CV 最直观的场景。

落地应用

车牌识别（LPR）—— >=99.5% 精度、违章检测（压实线、逆行、违停）—— 需要连续多帧分析、车流量统计 —— 城市交通信号灯优化、自动驾驶感知 —— 车道线/车辆/行人/交通标志检测。

技术特点

实时性要求极高（30+ FPS）、天气/光照/遮挡变化大、需要多任务模型（同时检测多个对象）、涉及跟踪（Tracking）和 ReID。

行业趣闻。 特斯拉 Autopilot 从 2016 年至今已经迭代了超过 10 版感知架构——从单一的 MobileNet 到 Occupancy Network + 端到端学习。每次架构变更背后都是对视觉理解能力的巨大考验。
智慧交通是 CV 最有"出圈"效应的场景。可以提一下 2025 年国内 L3 自动驾驶开始上量，这对视觉人才的需求有巨大拉动。但课程重点还是基础——先学会检测，再谈自动驾驶。

VR/AR 交互

行业场景 5/5

VR/AR 交互 · 理解世界才能增强世界

AR 的本质是在理解现实场景的基础上叠加虚拟信息。不理解场景，就无法做有意义的增强。

核心技术

技术详解

阶段 ① · 图像处理

阶段 1/5

图像处理 · 一切视觉任务的起点

在学习"智能"之前，先理解"视觉"本身。图像处理是后续所有任务的预处理基座。

核心操作

图像读取/写入（imread/imwrite）、颜色空间转换（BGR↔RGB↔HSV）、缩放/裁剪（resize）、仿射变换（旋转/平移/透视）、滤波降噪（高斯/中值/双边）、边缘检测（Canny/Sobel）、形态学操作（腐蚀/膨胀/开闭运算）。

为何重要

数据增强：随机裁剪/旋转/颜色抖动是深度学习训练的标配。预处理：归一化、去噪、对比度增强直接影响模型精度。结果可视化：在图片上画框、标文字是检测系统的基本交互方式。

一句话。 OpenCV 基础是训练数据预处理与结果可视化的前提。第一阶段会用 2-3 周时间让你熟练掌握 OpenCV 的核心 API。
这里强调 OpenCV 的重要性不是因为我们要在课程里花很多时间讲它，而是它是一个工具——后续每个阶段都会用到。比如 YOLO 训练时，数据增强就是通过 OpenCV 的仿射变换和颜色抖动实现的。建议同学课后看 OpenCV 官方教程：docs.opencv.org/4.x/d6/d00/tutorial_py_root.html。

阶段 2/5

深度学习基础 · 让机器学会"看"

传统 CV 靠人手工设计特征（SIFT、HOG、Haar），深度学习让网络自动学习特征。这是过去十年 CV 领域最大的范式转变。

CNN 核心概念

卷积层（Convolution）：学习局部特征（边缘→纹理→部件→物体）。池化层（Pooling）：降低特征图分辨率，增大感受野。全连接层（FC）：将特征映射到分类输出。激活函数（ReLU）：引入非线性，解决梯度消失问题。

PyTorch 视角

Tensor：PyTorch 的核心数据结构，类似 GPU 版的 NumPy ndarray。Autograd：自动求梯度——你只需写 forward，backward 自动计算。nn.Module：所有网络的基类，封装了参数管理和 forward 逻辑。

关键认知。 PyTorch 的 debug 流程和普通 Python 不同——你不能直接 print(tensor) 看中间值，需要用 .item() 或 .detach()。这是初学者最容易困惑的地方。
理解深度学习阶段最重要的三件事：卷积做特征提取、池化做降维、全连接做决策。PyTorch 中我们重点关注 nn.Conv2d、nn.MaxPool2d、nn.Linear 三个模块。课上我们会手写一个 5 层的 CNN 来分类手写数字——这个过程会让你彻底理解"端到端学习"的含义。

阶段 3/5

YOLO · 目标检测的工业化标准

YOLO（You Only Look Once）是目前工业界部署最广的目标检测架构。核心思想：将检测视为一个回归问题——单次前向传播同时输出边界框和类别概率。

YOLO 的发展

v1~v3：奠基（Joseph Redmon）→ v4/v5：工程化成熟 → v6/v7：效率优化 → v8：Ultralytics 官方维护，支持检测/分割/姿态/跟踪 → v9/v10：2024 最新架构改进。

YOLO 高速推理的结构原因

单阶段检测器——不需要两阶段方法（如 Faster R-CNN）的"先提候选框再分类"流程。YOLO 将图像划分为 S×S 网格，每个网格负责预测固定数量的候选框。T4 GPU 上 v8 可达 280+ FPS。

课程目标。 课程将训练 YOLOv8 安全帽检测模型作为核心案例，并最终部署为 Web 服务。整个过程约 80 行 Python 代码。
YOLO 是本课程的核心内容。这里只需要让同学有个概念：YOLO 为什么快、为什么大家都在用。具体的网络架构、loss 函数、训练技巧会后面专门讲。提一下 YOLO 的作者 Joseph Redmon 在 v3 之后退出 CV 领域的故事——因为担心技术被滥用。

常见错误根源

终端命令解析机制 —— 理解 PATH

学生最常见错误不是 Python 没装成功，而是终端找不到 Python：

# 错误信息，不是 Python 没装好
'python' 不是内部或外部命令，也不是可运行的程序
zsh: command not found: python
PATH 的作用
PATH 是操作系统查找可执行程序的路径列表。当你在终端输入 python，系统按 PATH 中列出的目录依次搜索名为 python (Windows) 或 python (macOS/Linux) 的可执行文件。
🔍 PATH 查看方式
Windows: echo %PATH% ｜ macOS/Linux: echo $PATH。PATH 是以 ;(Win) 或 :(Mac/Linux) 分隔的目录路径列表。
🛠️ PATH 修复方式
Windows: "系统属性 → 高级 → 环境变量" 添加 Python 路径。macOS: 检查 .zshrc 中是否有 export PATH="/Library/Frameworks/Python.framework/Versions/3.11/bin:$PATH"
PATH 问题是每年最常见的"假错误"。花 5 分钟讲清楚 PATH 原理，能解决 80% 的环境安装求助。建议在黑板上画一个示意图：终端输入 python → 系统遍历 PATH 列表 → 找到第一个 python.exe → 执行。如果遍历完没找到 → "command not found"。

工程原因

设计原理

虚拟环境 —— 三个关键工程问题

❶ 系统 Python 不适合作为项目环境

系统 Python 安装的包是全局共享的。项目 A 需要 OpenCV 4.8，项目 B 需要 OpenCV 4.5——全局装一个版本会导致另一个项目无法运行。虚拟环境为每个项目提供独立的 site-packages 目录，彻底隔离依赖冲突。

❷ 环境复现的标准做法

(.venv) $ pip freeze > requirements.txt
(.venv) $ cat requirements.txt
numpy==1.26.4
opencv-python==4.9.0.80
torch==2.2.2
...

别人拿到你的项目后，只需 pip install -r requirements.txt 即可安装完全相同的版本。这就是"可复现"。

❸ 虚拟环境与 Docker 的边界

虚拟环境只隔离 Python 包。Docker 隔离整个操作系统环境（包括系统库、环境变量、网络端口）。课程中我们用虚拟环境就足够了。Docker 在部署阶段介绍。

很多学生不理解"为什么要搞虚拟环境"。这个幻灯片回答了三个最常见的问题。重点强调：未来你的安全帽检测项目和别的同学的垃圾分类项目可能需要不同版本的 PyTorch——没有虚拟环境你会疯掉。

原因：Python 安装时未勾选"Add to PATH"。解决：重新安装并勾选 PATH 选项；或在环境变量中手动添加 Python 路径。

❌ pip: command not found

原因：Python 安装未包含 pip。解决：python -m ensurepip --upgrade 或重新安装 Python 并确保勾选 pip 组件。

❌ .\venv\Scripts\activate : 无法加载文件

原因：PowerShell 执行策略限制。解决：Set-ExecutionPolicy Unrestricted -Scope Process（仅当前会话）；或用 cmd 代替 PowerShell。

❌ pip install 报 SSL 错误 / 连接超时

原因：网络环境限制（常见于校园网/公司网络）。解决：使用镜像源 pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名。

❌ ImportError: DLL load failed

原因：Windows 缺少 VC++ 运行时。解决：安装 Microsoft Visual C++ Redistributable（从微软官网下载最新版）。

这页是实操环节的"求救中心"。建议打印出来贴在教室墙上。排错的基本思路：先确认错误信息（截图），再对照这个表找原因，最后按解决方案操作。如果全部尝试后依然无效——找助教/老师。不要盲目重复 pip install 十次。

工具链 1/3

数据标注工具

标注是监督学习的基础——没有标注数据，模型无法知道"这是安全帽"。三种主流的标注工具：

LabelImg

轻量级标注工具，基于 Python + Qt。支持矩形框 (YOLO/COCO 格式)。安装：pip install labelImg。适合小规模数据集（< 5000 张）。

Label Studio

功能更全面的开源标注平台，支持图像/文本/音频/视频多种类型。Web 界面，支持多人协作标注。安装：pip install label-studio。

Roboflow

在线标注平台（免费版 1000 张）。特色：内置数据增强、版本管理、一键导出多种格式。适合快速原型验证。

课程选择。 推荐使用 LabelImg 入门——安装简单、操作直观。后续批量标注会使用 Label Studio 的协作功能。Roboflow 作为备选方案。
这页开始讲工具链。标注是 CV 项目最耗时的环节——一个 2000 张的数据集，单人手标注大概需要 2-3 天。建议学生：先标注 100 张训练，用粗模型辅助标注剩余数据（半自动标注），再人工审核修正——这样可以节省 60% 的时间。

核心闭环

输入 → 处理 → 输出 → 保存

课堂演示

OpenCV 验证脚本 —— 第一个视觉"流水线"

创建一个 opencv_demo.py，理解视觉项目的基本闭环：

import cv2
import numpy as np

# 1. 创建一张空白图像（输入）
img = np.zeros((300, 500, 3), dtype=np.uint8)

# 2. 在图像上添加文字（处理）
cv2.putText(img, "OpenCV 环境验证 OK!",
            (80, 160), cv2.FONT_HERSHEY_SIMPLEX,
            1.2, (0, 255, 0), 3)

# 3. 绘制一个矩形框（检测框模拟）
cv2.rectangle(img, (50, 50), (200, 250),
              (255, 0, 0), 2)

# 4. 保存结果（输出）
cv2.imwrite("env_check.png", img)
print("✅ 已保存 env_check.png")
重点。 这不是一段简单的"画图"代码——它代表了所有视觉项目的基本模式：输入 → 处理 → 输出 → 保存。未来你做的任何视觉系统，底层逻辑都是这个闭环。
运行后让同学用图片浏览器打开 env_check.png。这是他们今天完成的第一个"视觉项目"。强调这个闭环思维比代码本身更重要：所有视觉系统都是"输入 → 处理 → 输出"的变体。安全帽检测系统本质上也是这样——输入一张图片 → 模型推理 → 输出检测结果。
