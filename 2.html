SECTION 1 · 图像基础

图像的本质：矩阵、张量与位深

数字图像在计算机中存储为多维 NumPy 数组。灰度图是 H×W 的二维矩阵，每个元素是像素亮度值。彩色图是 H×W×C 的三维张量，C=3 表示 B/G/R 三个颜色通道。

灰度图矩阵 (img.shape → (H, W))

32456890112145 405578105130160 387295120155190 7088110140175210

数组元素 img[y, x] 即 (行, 列) 处的像素值

彩色图张量 (img.shape → (H, W, 3))

每个像素位置有 3 个值，按 B/G/R 排列：

# 像素坐标 (y=2, x=3) 处的值
b = img[2, 3, 0]  # Blue  通道
g = img[2, 3, 1]  # Green 通道
r = img[2, 3, 2]  # Red   通道

# 切出单通道
blue_channel  = img[:, :, 0]  # (H, W) 矩阵
green_channel = img[:, :, 1]
red_channel   = img[:, :, 2]
位深 (bit depth)取值范围灰度级数dtype常见用途
8-bit0–255256uint8普通图像、JPEG、PNG
10-bit0–10231024uint16工业相机、RAW 格式
12-bit0–40954096uint16医疗影像 (DICOM)
16-bit0–6553565536uint16卫星遥感、HDR 图像

SECTION 1 · 通道顺序

BGR vs RGB：通道顺序陷阱与正确显示

OpenCV 使用 BGR 通道顺序（而非常见的 RGB），源于早期 BMP 格式的兼容性设计。若直接用 Matplotlib 显示 OpenCV 读取的彩色图像，红蓝通道互换，图像呈现蓝色偏。

正确的读取与显示流程

import cv2
import matplotlib.pyplot as plt

img = cv2.imread("photo.jpg")       # BGR

# 方法 1：转换通道顺序
img_rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
plt.imshow(img_rgb)

# 方法 2：手动翻转通道轴
img_rgb2 = img[:, :, ::-1]           # 反转第 3 维
# img[:, :, ::-1] 等价于 BGR→RGB

通道操作与 NumPy 索引

# 提取完整单通道
b = img[:, :, 0].copy()              # Blue  (H×W)

# 将红色通道全部置 0
img_no_red = img.copy()
img_no_red[:, :, 2] = 0

# BGR→RGB 另一种写法
rgb = img[..., ::-1]                 # ...=所有前导维度

# 不同格式的通道顺序
# OpenCV imread → BGR
# Matplotlib imshow → RGB
# PIL/Pillow → RGB
常见错误：cv2.imread() 后直接调用 plt.imshow(img) 会产生蓝色偏。校正方法：cv2.cvtColor(img, cv2.COLOR_BGR2RGB)。若写入文件，cv2.imwrite() 始终使用 BGR 顺序，无需转换。

SECTION 2 · 色彩空间

BGR、灰度、二值、HSV 四大色彩空间

同一幅图像可用不同色彩空间表示。每种空间强调信息的不同侧面，适用于不同的处理任务。色彩空间转换使用统一的 cv2.cvtColor() 函数。

色彩空间通道数维度含义OpenCV 转换标志适用场景
BGR / RGB3蓝/绿/红 强度— (原生)图像读取、显示、存储
GRAY1亮度值 [0,255]COLOR_BGR2GRAY边缘检测、特征提取、形态学
BINARY1仅 0 或 255threshold()掩膜、轮廓提取
HSV3色相/饱和度/明度COLOR_BGR2HSV颜色分割、目标跟踪

灰度化的数学原理：亮度加权公式

灰度化并非简单取 RGB 均值。人眼对绿色最敏感、蓝色最不敏感，权重分配基于 ITU-R BT.601 标准：

Gray = 0.299 × R + 0.587 × G + 0.114 × B

0.299

红色 — M 视锥细胞中等敏感

0.587

绿色 — M 视锥细胞最多

0.114

蓝色 — S 视锥细胞最少

生物学背景：人类视网膜含三种视锥细胞：L（红，约 64%）、M（绿，约 32%）、S（蓝，约 2%）。加权公式模拟了视锥细胞的分布比例。

SECTION 2 · 决策指南

何时灰度化，何时保留颜色

灰度化压缩颜色信息，仅保留亮度结构。以下决策树帮助判断是否需要灰度化——如果颜色本身包含关键信息，不可盲目灰度化。

可灰度化的场景

  • 边缘检测：Canny / Sobel 只需亮度梯度
  • 特征匹配：SIFT / ORB 基于梯度直方图，颜色不参与
  • 形状分析：轮廓提取、多边形拟合与颜色无关
  • 纹理分析：LBP / GLCM 计算灰度共生矩阵
  • 光流法：亮度恒定假设下估计运动向量
  • 模板匹配：matchTemplate 在灰度图上更稳定

必须保留颜色的场景

  • 交通灯检测：红色/绿色是核心判据
  • 果实成熟度：颜色变化指示成熟阶段
  • 工业色差检测：颜色差异 = 缺陷指标
  • 基于 HSV 的分割：利用色相区分目标
  • 多光谱分析：远红外/近红外等多通道
  • 语义分割/实例分割：颜色是重要特征
设计原则：灰度化应视为"当亮度结构足够完成任务时的优化步骤"，而非默认始终执行的预处理。每次使用前需明确判断颜色信息是否包含任务所需的关键辨別维度。

SECTION 2 · 二值图与阈值分割

二值化：从灰度到 0/255 的阈值分割

二值图 (Binary Image) 的像素值仅为 0 或 255。通过阈值分割从灰度图得到。常用于生成掩膜 (Mask) 标记感兴趣区域。

固定阈值：cv2.threshold

# 阈值 = 127, 最大 = 255
_, bin = cv2.threshold(
    gray, 127, 255, cv2.THRESH_BINARY
)

# 四种阈值模式
cv2.THRESH_BINARY      # 标准二值化
cv2.THRESH_BINARY_INV  # 反转二值化
cv2.THRESH_TRUNC       # 截断: v>thr=thr
cv2.THRESH_TOZERO      # 归零: v

自适应阈值（光照不均场景）

binary = cv2.adaptiveThreshold(
    gray, 255,
    cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
    cv2.THRESH_BINARY,
    blockSize=11,    # 邻域大小（奇数）
    C=2              # 常数减项
)

# ADAPTIVE_THRESH_GAUSSIAN_C
# 使用高斯加权均值作为局部阈值
# C 为正则化常数: T = mean - C
阈值选型指南：全局照明均匀 → 固定阈值 (速度快)；光照渐变或阴影 → 自适应阈值（更精确，但计算更慢）。blockSize 值越大，对局部光照变化越不敏感。

SECTION 2 · HSV 应用

基于 HSV 的颜色分割与范围调优

HSV 最典型应用：通过色相 (H) 范围提取特定颜色目标。相比 BGR 空间为每个通道分别设阈值，HSV 在光照变化条件下更稳定。

蓝色目标提取

hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)

# 蓝色 H 范围 [100, 130]
lower = np.array([100, 50, 50])
upper = np.array([130, 255, 255])

mask = cv2.inRange(hsv, lower, upper)
result = cv2.bitwise_and(
    img, img, mask=mask
)

SECTION 3 · 噪声

图像噪声的来源分类与仿真

图像噪声是采集、传输、压缩过程中引入的随机干扰。噪声的类型决定了应选用何种滤波方法。认识噪声是选择滤波器的先决条件。

噪声类型产生原因数学特性适用滤波
高斯噪声传感器热噪声，低照度放大正态分布 N(0, σ²)高斯滤波 / 均值滤波
椒盐噪声传输误码，ADC 采样错误随机黑白像素点 (0 或 255)中值滤波
泊松噪声光子计数统计波动泊松分布，与信号强度相关高斯滤波 + 方差稳定变换
相干斑噪声激光 / 超声成像干涉乘性噪声 (speckle)Lee / Kuan 滤波
# NumPy 噪声仿真（帮助理解噪声视觉特征）
img = cv2.imread("sample.jpg", 0)                    # 灰度

# 高斯噪声：np.random.normal(mean, std, shape)
gauss = np.random.normal(0, 25, img.shape).astype(np.int16)
noisy_gauss = np.clip(img.astype(np.int16) + gauss, 0, 255).astype(np.uint8)

# 椒盐噪声：随机分布的黑/白点
sp = img.copy()
mask = np.random.random(img.shape)
sp[mask < 0.02]  = 0      # 2% 概率置黑
sp[mask > 0.98]  = 255    # 2% 概率置白

SECTION 3 · 均值滤波

均值滤波：原理、实现与边缘模糊问题

均值滤波 (Mean / Box Filter) 是最简单的线性滤波。以目标像素为中心，k×k 窗口内所有像素的算术平均值作为输出。每个邻域像素权重相等。

I'(x,y) = (1/k²) · ΣΣ I(i,j) (i,j ∈ k×k 邻域)

3×3 均值核可视化

1/91/91/9 1/91/91/9 1/91/91/9

所有权重均为 1/9，归一化后和为 1.0

OpenCV 实现

# ksize = 5（必须奇数）
blur = cv2.blur(img, (5, 5))

# 等价写法
blur2 = cv2.boxFilter(
    img, -1, (5, 5),
    normalize=True
)
边缘模糊的根源：窗口跨越边缘时，高值区域（物体）和低值区域（背景）被平均，破坏锐利过渡。ksize 越大，参与平均的像素越多，模糊越严重。适用：对边缘保留要求不高、计算速度优先的场景（如实时视频预处理器）。

SECTION 3 · 高斯滤波

高斯滤波：数学原理与参数含义

高斯滤波使用高斯分布权重卷积核：中心像素权重最大，距离越远权重按钟形曲线递减。二维高斯函数如下：

G(x,y) = (1 / 2πσ²) · exp(−(x² + y²) / 2σ²)

5×5 高斯核 (σ=1, 整数近似)

14741 41626164 72641267 41626164 14741

中心 (41) 权重远超角落 (1)，中心主导输出

σ 对核形状的影响

# σ 小 → 核集中在中心
#   (保留更多细节)
# σ 大 → 核分布更均匀
#   (平滑更强)
# 经验法则：
# ksize ≈ 6σ + 1
# σ=1.0 → ksize=7
# 高斯核权重分布示例（手动生成验证）
import cv2, numpy as np

# 查看 5×5 高斯核的真实数值（σ=1.0）
kernel = cv2.getGaussianKernel(5, 1.0)
gauss_2d = kernel @ kernel.T         # 外积 = 二维核
print(gauss_2d.round(3))

中值滤波：medianBlur——椒盐噪声克星

中值滤波将窗口像素的中位数替代中心值。不是线性卷积，而是排序统计滤波，能保留边缘的同时去除椒盐噪声。

median = cv2.medianBlur(img, 5)

# 3×3: 保留细节，去噪有限
# 7×7: 去噪强，但细节损失大

示例：窗口 [0, 128, 130, 132, 255, 126, 129, 131, 127] → 排序后中位数 = 129。极值 0 和 255 被跳过。

从传统滤波到深度学习卷积 (CNN)

传统图像滤波与 CNN 卷积在数学形式上完全一致（滑动窗口 · 加权求和），核心区别在于卷积核的来源：

传统滤波 (OpenCV)

  • 卷积核人工设计（高斯、均值、Sobel）
  • 数学原理明确、可解释性强
  • 参数极少（仅 ksize, sigma）
  • 计算速度快，不需训练
  • 泛化能力有限，每类场景需手工调参

CNN 卷积 (深度学习)

  • 卷积核从数据中学习获得
  • 自动提取任务相关特征
  • 参数量大（常以百万计）
  • 需要 GPU 加速训练
  • 需大量标注数据

二者不是替代关系。实践中常组合使用：传统方法 → 预处理降噪 CNN → 高层语义分析。例如：先用高斯滤波去除传感器噪声，再输入 YOLO / ResNet 做目标检测。

常见误解：CNN 的"卷积层"名称来源于图像处理中的卷积概念。理解传统卷积的滑动窗口机制，是理解 CNN 感受野、池化、空洞卷积的基础。

SECTION 4 · 几何变换

几何变换：坐标系统的数学映射

几何变换 (Geometric Transformation) 不是改变图像显示尺寸，而是重新映射像素坐标。每个输出像素从输入图像中对应位置插值得到。变换的五个级别：

变换矩阵维度自由度OpenCV 函数保持性质
缩放 (Scale)2×32resize()形状
平移 (Translate)2×32warpAffine()形状、大小
旋转 (Rotate)2×31warpAffine()形状 (绕中心)
仿射 (Affine)2×36warpAffine()平行 + 共线
透视 (Perspective)3×38warpPerspective()共线 (直线仍直线)

核心原则：变换坐标 ≠ 变换像素

若图像经历几何变换，目标检测的标注框坐标必须同步变换——框的坐标用相同的变换矩阵计算，不是手动调整偏移。

# warpAffine 核心参数
dst = cv2.warpAffine(
    src,        # 输入图像
    M,          # 2×3 变换矩阵
    dsize,      # 输出尺寸 (width, height)
    flags=cv2.INTER_LINEAR,         # 插值方法
    borderMode=cv2.BORDER_CONSTANT  # 边界填充策略
)

# 3. 指定插值

    img, (0, 0),

方法适用速度

SECTION 4 · 平移 & 旋转

平移与旋转：仿射变换的两种基本形式

平移和旋转是最基础的两类仿射变换。两者都可用 2×3 矩阵表示，且都通过 cv2.warpAffine() 执行。

平移：矩阵与代码

[x'] = [1 0 tx][x]
[y'] [0 1 ty][y]

h, w = img.shape[:2]
M = np.float32([
    [1, 0, 100],      # 右移 100px
    [0, 1,  50]       # 下移  50px
])
shifted = cv2.warpAffine(
    img, M, (w, h)
)

绕中心旋转 + 画布适配

center = (w//2, h//2)
M = cv2.getRotationMatrix2D(
    center, 45, 1.0
)
# 自动计算新画布尺寸
cos, sin = abs(M[0,:2])
nw = int(h*sin + w*cos)
nh = int(h*cos + w*sin)
M[:,2] += [nw/2-center[0],
           nh/2-center[1]]
rotated = cv2.warpAffine(
    img, M, (nw, nh)
)

绕中心旋转 = 先平移到原点 → 旋转 → 平移到新中心。cv2.getRotationMatrix2D 自动完成这个复合变换计算。

SECTION 4 · 仿射与透视

仿射变换 (3 点) vs 透视变换 (4 点)

仿射：保持平行性，2×3 矩阵，3 组对应点求解。透视：平行线可能相交（近大远小），3×3 矩阵，4 组对应点求解。

仿射变换 (getAffineTransform)

src = np.float32([
    [50,50],[200,50],[50,200]
])
dst = np.float32([
    [10,100],[200,50],[100,250]
])
M = cv2.getAffineTransform(src, dst)
affined = cv2.warpAffine(
    img, M, (w, h)
)

透视变换 (getPerspectiveTransform)

src = np.float32([
    [150,200],[650,150],
    [700,800],[100,750]
])
dst = np.float32([
    [0,0],[600,0],
    [600,800],[0,800]
])
M = cv2.getPerspectiveTransform(src, dst)
warped = cv2.warpPerspective(
    img, M, (600, 800)
)
典型应用：仿射 → 旋转校正、图像对齐、数据增广。透视 → 文档扫描矫正、车道线鸟瞰图 (BEV)、广告牌替换。透视变换的精度高度依赖对应点选取的准确性。

SECTION 5 · 综合管线

图像预处理管线：从传感器到模型输入

目标检测管线中，预处理将原始传感器数据转换为检测模型可以稳定处理的标准化输入。

STEP 1图像读取与尺寸统一 — cv2.imread() → resize(640×640)，确保批次内尺寸一致
STEP 2色彩空间转换 — 灰度模型 → BGR2GRAY；彩色模型 → BGR2RGB（匹配训练数据格式）
STEP 3噪声抑制 — 高斯滤波 (5×5, σ=1.2) 消除传感器热噪声，提升边缘稳定性
STEP 4几何校正 — 透视变换矫正拍摄角度，使目标在标准视角下被检测
STEP 5归一化 — 像素值 [0,255] → [0,1] 或 [-1,1]（浮点数），加速模型收敛
# 完整预处理函数（可复用于任何推理任务）
def preprocess(image_path, input_size=640):
    img = cv2.imread(image_path)
    img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
    img = cv2.GaussianBlur(img, (5,5), 1.2)
    img = cv2.resize(img, (input_size, input_size))
    img = img.astype(np.float32) / 255.0   # [0,1]
    return img

resized = cv2.resize(blurred, (640, 480))              # ③

cv2.imwrite("02_blurred.jpg", blurred)
cv2.imwrite("03_resized.jpg", resized)

# 步骤 6-10：滤波对比 → 几何变换 → Canny 边缘 → 保存
mean  = cv2.blur(gray, (5, 5))                         # ④ 均值
gauss = cv2.GaussianBlur(gray, (5, 5), 1.2)             # ⑤ 高斯
median = cv2.medianBlur(gray, 5)                        # ⑥ 中值

h, w = gray.shape
M = cv2.getRotationMatrix2D((w//2, h//2), 15, 1.0)
rotated = cv2.warpAffine(gray, M, (w, h))               # ⑦ 旋转
small = cv2.resize(rotated, (0,0), fx=0.5, fy=0.5)     # ⑧ 缩放

edges_blur = cv2.Canny(gauss, 50, 150)                  # ⑩

for name, data in [("mean",mean),("gauss",gauss),("median",median),
                 ("rotated",rotated),("small",small),
                 ("edges_raw",edges_raw),("edges_blur",edges_blur)]:
    cv2.imwrite(f"04_{name}.jpg", data)
print("预处理完成 — 共 10 个中间文件")

本讲总结

图像表示

灰度图 H×W 矩阵，彩色图 H×W×C 张量。OpenCV 默认 BGR 顺序。dtype = uint8 (0-255)。

灰度化

加权公式 Gray = 0.299R + 0.587G + 0.114B。使用 cv2.cvtColor(BGR2GRAY)。颜色是否保留取决于任务需求。

色彩空间

HSV 按色相分割颜色，适合颜色目标提取。BGR 用于设备原生操作。转换用统一的 cvtColor 接口。

滤波

均值 blur 简单但模糊边缘。高斯 GaussianBlur 保边中。中值 medianBlur 对椒盐噪声最有效。本质 = 卷积。

几何变换

缩放/平移/旋转/仿射/透视。warpAffine (2×3) 或 warpPerspective (3×3)。检测框坐标必须同步变换。

预处理管线

读取→尺寸统一→色彩转换→降噪→几何校正→归一化。目标是消除与任务无关的变量。
