§ 1.1 · 现实背景

为什么需要少样本目标检测

常规目标检测方法（如 Faster R-CNN、YOLO、SSD）依赖大量标注样本。每个新类别通常需要数千张标注图像才能达到可用精度。然而在真实工业场景中，这一假设经常无法满足。

5%
工业缺陷检测中，缺陷样本占总产量的比例通常低于 5%
1-10
新品类上线初期，可用标注样本往往仅有 1-10 张
80%
长尾分布中，80% 的类别属于样本极少的尾部类别

少样本目标检测（Few-shot Object Detection, FSOD）正是在此背景下提出：给定极少量新类别标注样本，使模型具备检测这些新类别目标的能力。

§ 1.2 · 典型场景分析

样本稀缺的行业场景

工业质检
  • 新产品产线刚投产，缺陷样本极少
  • 缺陷类型随工艺调整不断变化
  • 标注需要领域专家，成本高昂
医疗影像
  • 罕见病例的影像数据极其有限
  • 隐私法规限制数据共享与标注
  • 病灶形态多样，标注一致性难保证
安全监控
  • 安全事故场景（火灾、交通事故）难以采集
  • 异常事件发生频率极低
  • 场景变化快，固定模型迅速过时
卫星遥感
  • 特定目标（稀有船只、临时建筑）样本少
  • 标注需要专业判读员
  • 类别变化快，来不及大规模标注
[共性] 上述场景的共同特征：类别变化快、标注成本高、长尾分布显著。少样本检测并非替代常规检测，而是填补"样本不足"这一空白区域。

§ 1.3 · 问题定位

常规检测与少样本检测的差异

维度常规目标检测少样本目标检测
数据假设每类数百至数千标注样本新类每类仅 1/5/10 个样本
训练方式端到端训练或标准迁移学习基类预训练 + 少样本微调
核心挑战数据质量、调参、部署效率过拟合、类别混淆、泛化能力
评估重点mAP@0.5:0.95、推理速度新类 mAP、基类知识保留度
典型方法YOLO、Faster R-CNN、DETRFsDet、TFA、FSCE、DeFRCN

少样本检测的核心矛盾：模型需要从极少量样本中学习新类别的视觉概念，同时不能遗忘已学会的基类知识。这要求训练策略在设计上做出针对性调整。

§ 2.1 · 核心术语

Base Class · Novel Class · K-shot

Base Classes（基类）
  • 训练集中拥有大量标注样本的类别
  • 模型在基类上充分训练，获得通用检测能力
  • 典型数量：每类数百至数千张图像
  • 例如 COCO 数据集中的 60 个基类
Novel Classes（新类）
  • 只有少量标注样本的全新类别
  • 模型需要从极少量样本中学会检测
  • 与基类不重叠，构成"新"的检测目标
  • 例如 COCO 中预留的 20 个新类
K-shot 设定
  • K 表示每个新类可用的标注实例数量
  • 常见设定：1-shot、3-shot、5-shot、10-shot
  • K 值越小，任务难度越大，过拟合风险越高
  • K-shot 中的"shot"指标注的边界框实例，而非图像张数
[示例] 5-shot 设定：新类"猫"有 5 张标注图像，每张图像中可能包含多个猫实例，但"shot"计数以实例（边界框）为单位。

§ 2.2 · 数据划分

Support Set 与 Query Set

Support Set（支持集）
  • 少量带标注的样本，用于"告诉"模型新类长什么样
  • 每个新类包含 K 个标注实例（边界框 + 类别标签）
  • 在微调阶段作为训练数据使用
  • 支持集的质量直接影响微调效果
Query Set（查询集）
  • 待检测的图像集合，用于评估模型性能
  • 包含新类和基类的目标实例
  • 模型需要在查询集上检测出所有已知类别
  • 评估指标：新类 mAP、基类 mAP、总体 mAP

[关键] Support set 和 query set 的类别不重叠：support set 只包含新类样本，query set 同时包含基类和新类。这模拟了真实场景——已知类别有充足数据，新类别只有少量样本。

§ 2.3 · 任务对比

少样本分类 vs 少样本检测

维度少样本分类少样本目标检测
任务定义判断图像属于哪个类别定位目标位置 + 判断类别
输出类别标签（单标签或多标签）边界框集合 + 类别标签 + 置信度
额外挑战类别判别定位精度、背景抑制、多目标
典型方法MAML、ProtoNet、RelationNetFsDet、TFA、FSCE、DeFRCN
难度评估中等显著更高（定位 + 分类联合学习）

少样本检测的额外难度来源于：模型不仅要判断"这是什么"，还要回答"其空间位置"。在样本极少的情况下，定位分支比分类分支更容易过拟合，因为边界框回归对数据量更敏感。

[延伸] 少样本检测可视为"少样本分类 + 目标定位"的组合问题。FsDet 的核心思路是：利用基类训练让定位能力充分学习，新类只需微调分类能力。

§ 2.4 · 概念体系总结

术语体系一览

数据维度
  • Base classes：基类，大量标注样本可用
  • Novel classes：新类，仅 K 个标注实例
  • K-shot：每个新类的标注实例数量
  • Support set：K-shot 新类样本集合
  • Query set：待检测的测试图像集合
训练维度
  • Base training：在基类上训练通用检测器
  • Few-shot fine-tuning：用新类样本微调
  • Novel set evaluation：评估新类检测精度
  • Base+Novel evaluation：评估全部类别精度
[记忆] Base = 已知大量；Novel = 未知少量；Shot = 样本计数单位；Support = 训练用；Query = 测试用。

FsDet 对应论文 "Frustratingly Simple Few-Shot Object Detection"（ICML 2020），由 UC Berkeley 团队提出。核心思想可概括为一句话：不设计复杂的新结构，而是通过合理的训练策略让标准检测器适应少样本场景。

通用检测能力
→
Few-shot FT

  • 提供标准化的少样本检测评估协议
  • 开源实现，便于复现和扩展

§ 3.2 · 两阶段训练逻辑

Stage 1: Base Training

第一阶段使用大量基类数据训练检测器，使其掌握通用视觉检测能力。此阶段与常规目标检测训练完全相同。

模型在此阶段学习：
  • 如何从图像中提取通用视觉特征（backbone）
  • 什么区域可能是物体（RPN 候选区域生成）
  • 如何回归精确的边界框（bbox regression）
  • 如何区分类别（classification head）
训练配置示例：
# Base training 关键参数 SOLVER.IMS_PER_BATCH 16 SOLVER.BASE_LR 0.02 SOLVER.MAX_ITER 90000 SOLVER.STEPS (60000, 80000) MODEL.ROI_HEADS.NUM_CLASSES 60 # 基类数
[关键] Base training 的输出是一个在基类上表现良好的检测器。其 backbone 和 RPN 学到的通用视觉能力（边缘检测、纹理识别、物体性判断）对后续新类学习至关重要。

§ 3.3 · 两阶段训练逻辑

Stage 2: Few-shot Fine-tuning

第二阶段使用少量新类样本对基类检测器进行微调。这是 FsDet 的核心环节，也是最容易出现问题的环节。

关键难点：
  • 过拟合：样本极少（5-10 张），模型容易记住而非泛化
  • 类别混淆：新类与基类在特征空间可能重叠
  • 灾难性遗忘：过度微调会破坏基类学到的能力
  • 样本偏差：少量样本可能不代表真实分布
微调配置示例：
# Few-shot fine-tuning 关键参数 SOLVER.IMS_PER_BATCH 4 SOLVER.BASE_LR 0.001 # 降低学习率 SOLVER.MAX_ITER 1000 # 少量迭代 SOLVER.STEPS (600, 800) MODEL.ROI_HEADS.NUM_CLASSES 80 # 基类+新类 # 冻结 backbone 前两层 MODEL.BACKBONE.FREEZE_AT 2

§ 3.4 · 微调策略详解

应对少样本微调的关键策略

策略一：降低学习率
  • 基类训练学习率：0.02
  • 微调学习率：0.001（降低 20 倍）
  • 目的：防止大步长更新破坏已学特征
策略二：冻结部分 Backbone
  • 冻结 backbone 底层（如 ResNet 前两个 stage）
  • 底层学习的是通用特征（边缘、纹理），无需调整
  • 高层特征和检测头需要适应新类
策略三：限制迭代次数
  • 基类训练：90,000 次迭代
  • 微调：1,000-2,000 次迭代
  • 目的：防止过拟合和灾难性遗忘
策略四：多次随机采样
  • 从 support set 中多次随机采样不同组合
  • 分别微调后取平均结果
  • 缓解样本偏差带来的结果波动

§ 3.5 · 微调策略效果对比

不同微调策略的效果分析

策略新类 mAP基类 mAP说明
全参数微调（高学习率）低显著下降灾难性遗忘严重，基类能力被破坏
全参数微调（低学习率）中轻微下降基类保留较好，但新类学习不充分
冻结 backbone + 微调 head高保持最佳平衡：保留通用特征，调整分类能力
仅微调分类头中高保持最保守策略，适合 1-shot 极端场景
多次采样平均高（稳定）保持增加计算开销但结果更可靠

FsDet 论文的实验表明：冻结 backbone 前两层 + 低学习率微调 ROI head 是最优组合。这一配置在保留基类能力的同时，使新类 mAP 达到最高。

[结论] 少样本微调的本质是在"学习新类"和"不忘旧类"之间寻找平衡。FsDet 的贡献在于系统性地找到了这个平衡点，而非发明新的网络结构。

  • RPN：生成候选目标区域（Region Proposals）
  • ROI Align：将候选区域映射为固定尺寸特征
  • ROI Head：对每个候选区域分类 + 边界框回归
少样本场景下的角色
  • Backbone + RPN：从基类学到通用能力，微调时通常冻结
  • ROI Head：分类头需要适应新类，是微调的主要对象
  • 这种分工使少样本微调成为可能

§ 4.2 · Backbone 与 RPN

特征提取与候选区域生成

Backbone（特征提取网络）
  • 常用结构：ResNet-50、ResNet-101
  • 配合 FPN（特征金字塔网络）实现多尺度检测
  • 输入：3×H×W 图像 → 输出：多尺度特征图
  • 在少样本场景中，backbone 从基类充分训练
  • 微调时冻结底层（保留通用特征），可选微调高层
# 冻结 backbone 前 2 个 stage MODEL.BACKBONE.FREEZE_AT 2
RPN（区域建议网络）
  • 在特征图上滑动窗口，生成候选区域
  • 每个位置预设 K 个 anchor box（不同尺寸和比例）
  • 输出：objectness score + 边界框修正
  • NMS 后保留 Top-N 候选区域
  • 少样本场景中 RPN 通常不微调——"找物体"的能力是通用的
# RPN anchor 配置 MODEL.RPN.ANCHOR_SIZES (32,64,128,256,512) MODEL.RPN.ASPECT_RATIOS (0.5,1.0,2.0) MODEL.RPN.PRE_NMS_TOPK_TEST 6000 MODEL.RPN.POST_NMS_TOPK_TEST 1000

§ 4.3 · ROI Align 与 ROI Head

候选区域处理与预测

ROI Align
  • 将 RPN 输出的候选区域（不同尺寸）映射为固定尺寸特征图
  • 使用双线性插值，避免 ROI Pooling 的量化误差
  • 输出尺寸固定（如 7×7 或 14×14）
  • 对检测小目标至关重要（保留空间精度）
# ROI Align 配置 MODEL.ROI_HEADS.BATCH_SIZE_PER_IMAGE 512 MODEL.ROI_BOX_HEAD.POOLER_RESOLUTION 7 MODEL.ROI_BOX_HEAD.POOLER_SAMPLING_RATIO 2
ROI Head（检测头）
  • 对每个候选区域执行：
  • 分类分支：预测类别（C+1 类，含背景）
  • 回归分支：预测边界框偏移量（4×C 个值）
  • 少样本微调的主要对象
  • 分类头需要扩展维度以包含新类
# ROI Head 配置 MODEL.ROI_HEADS.NUM_CLASSES 80 # 基类+新类 MODEL.ROI_BOX_HEAD.FC_DIM 1024 MODEL.ROI_BOX_HEAD.NUM_FC 2

§ 4.4 · 少样本场景的适配策略

Faster R-CNN 在少样本场景中的适配

FsDet 对标准 Faster R-CNN 的修改非常有限，这也是其"Frustratingly Simple"名称的由来。主要修改集中在训练策略而非网络结构。

结构层面的修改
  • 分类头维度扩展：从 C_base 扩展到 C_base + C_novel
  • 新增类别的分类权重随机初始化（或基类权重复制）
  • 其余网络结构完全不变
# 分类头维度变化 # Base training: 60 类 MODEL.ROI_HEADS.NUM_CLASSES 60 # Fine-tuning: 60 + 20 = 80 类 MODEL.ROI_HEADS.NUM_CLASSES 80
训练策略的修改
  • 学习率降低 20 倍（0.02 → 0.001）
  • 迭代次数减少 90 倍（90000 → 1000）
  • 冻结 backbone 底层（FREEZE_AT=2）
  • 使用加权采样平衡基类和新类
[启示] FsDet 的设计哲学：在数据极度稀缺时，复杂的网络结构设计不如合理的训练策略有效。这一思想对少样本学习的各个领域都有借鉴意义。

§ 6.1 · 少样本检测的局限

少样本检测不是魔法

必须明确：少样本检测并不能在所有场景中替代常规检测。理解其局限性，才能做出正确的技术选型。

核心局限
  • 新类与基类差异过大时效果差：如果新类的视觉特征与基类完全不同，backbone 提取的特征无法有效表达新类
  • 样本偏差问题：K-shot 样本如果不代表真实分布，模型会产生严重偏置
  • 小目标检测困难：小目标在特征图中占据像素少，少量样本不足以学习有效特征
  • 遮挡与复杂背景：样本量少导致模型对遮挡和背景干扰的鲁棒性不足
工程实践中的挑战
  • 结果波动大：不同随机种子、不同 support set 采样导致结果差异显著
  • 需要多次实验：单次微调结果不可靠，需要多次采样取平均
  • 部署复杂度高：两阶段训练流程比端到端检测复杂，工程维护成本高
  • 基类性能可能下降：即使精心调参，微调后基类 mAP 通常会有 1-3% 的下降
[建议] 在项目中，FsDet 适合作为扩展模块或研究体验。如果新类样本可以积累到每类 50+ 张，常规检测或标准迁移学习通常是更稳定、更简单的选择。

维度YOLO 常规训练FsDet 少样本检测

§ 6.4 · 课外拓展与总结

拓展阅读与思考方向

进阶方法阅读
  • TFA（ICLR 2020）：两阶段微调，先训练全部参数再仅微调分类头
  • FSCE（CVPR 2021）：引入对比学习约束增强新类特征判别性
  • DeFRCN（NeurIPS 2021）：解耦 RPN 与 ROI Head 梯度传播
  • Meta R-CNN（ICLR 2019）：基于元学习的少样本检测方法
思考方向
  • 样本极少时，可比较继续标注、少样本方法与开放词表模型（如 GLIP、Grounding DINO）的适用条件。
  • 少样本检测与零样本检测（Zero-shot Detection）的本质区别。
  • 多模态大模型替代少样本检测的可行性边界。
核心要点回顾
  • FsDet 的核心思想：强基线 + 合理训练策略，无需复杂结构
  • 两阶段流程：base training → few-shot fine-tuning
  • 微调关键：低学习率、冻结 backbone、限制迭代、多次采样
  • 适用边界：1-10 shot 极端少样本场景，非通用替代方案
