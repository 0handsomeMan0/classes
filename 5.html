目标检测任务定义与数学形式

目标检测（Object Detection）是计算机视觉的核心任务，要求模型同时解决两个子问题：目标定位（确定目标在图像中的空间位置）和目标分类（判断目标所属类别）。与图像分类不同，检测任务需要输出可变数量的检测结果，每张图像中目标的数量和位置均未知。

数学形式化定义：给定输入图像 I ∈ ℝH×W×3，检测模型输出检测结果集合 D = {d1, d2, ..., dN}，每个 di = (x, y, w, h, c, s) 包含边界框中心坐标、宽高、类别标签和置信度分数。

关键挑战

目标尺度差异大（小目标仅数十像素）；密集遮挡导致边界模糊；光照变化与形变；实时性要求（视频流需≥30 FPS）；类别不平衡（背景远多于前景）。

核心评估指标

mAP（mean Average Precision）是检测精度核心指标。IoU阈值通常设为0.5（mAP@0.5）或0.50:0.95（mAP@0.5:0.95，COCO官方指标）。FPS衡量推理速度。

# 边界框标注格式与IoU计算（核心评价指标） def compute_iou(box1, box2): # box: [x1, y1, x2, y2] 左上角+右下角坐标 x1 = max(box1[0], box2[0]); y1 = max(box1[1], box2[1]) x2 = min(box1[2], box2[2]); y2 = min(box1[3], box2[3]) inter = max(0, x2 - x1) * max(0, y2 - y1) area1 = (box1[2]-box1[0]) * (box1[3]-box1[1]) area2 = (box2[2]-box2[0]) * (box2[3]-box2[1]) return inter / (area1 + area2 - inter) # IoU = 交集/并集

两阶段 先粗后精

第一阶段生成候选区域（Region Proposals），第二阶段逐区域分类回归。精度高，速度慢。代表：Faster R-CNN（~5-10 FPS）。

消除锚框和NMS等手工设计组件。全局感受野，易于扩展到开放词表检测。代表：DETR、DINO。

两阶段检测器：R-CNN → Fast → Faster

两阶段检测器的核心思想是"先粗后精"：第一阶段生成候选区域，第二阶段对每个区域精细分类和回归。这种解耦设计使每个阶段可专注于各自任务。

R-CNN（2014）

Selective Search生成约2000候选区域，逐区域缩放后送入CNN分类。重复计算量大，训练分多个阶段。推理约47秒/张。

Fast R-CNN（2015）

引入RoI Pooling层，整图仅过一遍CNN，在特征图上提取候选区域特征。使用多任务损失联合训练，速度提升约10倍（~2秒/张）。

Faster R-CNN（2015）

用RPN（Region Proposal Network）替代Selective Search。RPN在特征图滑动锚框，输出前景/背景分类和粗略框回归。真正端到端训练，速度约0.2秒/张。

RPN 工作原理

RPN在特征图每个位置放置 k 个锚框（通常 k=9，对应3种尺度×3种长宽比）。对每个锚框输出二分类分数和4个回归偏移 (Δx, Δy, Δw, Δh)。正负样本按IoU划分：IoU > 0.7为正样本，IoU < 0.3为负样本。

关键设计思想

RPN与检测Head共享Backbone特征，几乎不增加额外计算量。这是两阶段范式从外部算法（Selective Search）到可学习网络模块的关键演进。后续RoI Align解决了RoI Pooling的量化误差问题，在Mask R-CNN中首次提出。

一阶段检测器：YOLO / SSD / RetinaNet

一阶段检测器不显式生成候选区域，而是在特征图上直接预测类别和边界框。这种密集预测策略将检测视为统一的回归问题，大幅提升推理速度。

YOLO（2016）

图像划分为 S×S 网格，每个网格预测 B 个边界框和 C 个类别。统一回归框架，速度45-150 FPS。早期版本（v1/v2）对小目标和密集目标检测效果较差。

SSD（2016）

在多个尺度的特征图上设置锚框，直接预测类别分数和框偏移。VGG16作为Backbone，额外卷积层生成多尺度特征图。小目标检测优于YOLOv1，速度与YOLO相当。

RetinaNet（2017）

提出Focal Loss解决正负样本极端不平衡问题。Focal Loss降低易分类负样本的损失权重，使模型聚焦于难样本。首次使一阶段检测器精度超越Faster R-CNN。

# Focal Loss 公式与实现 # FL(p_t) = -α_t · (1 - p_t)^γ · log(p_t) # 当 γ=0 时退化为标准交叉熵损失；通常取 γ=2, α=0.25 def focal_loss(pred, target, gamma=2.0, alpha=0.25): pt = torch.where(target == 1, pred, 1 - pred) # 对正/负样本分别计算 focal_weight = (1 - pt) ** gamma if alpha is not None: focal_weight = focal_weight * (alpha * target + (1 - alpha) * (1 - target)) return -focal_weight * pt.log()

Transformer检测器：DETR 与 DINO

DETR（Detection Transformer）由Facebook AI于2020年提出，将Transformer架构引入检测，消除了对锚框、RPN和NMS等手工设计组件的依赖，实现端到端检测。

DETR 核心组件

CNN Backbone提取图像特征后展平为序列；Transformer Encoder通过自注意力建模全局特征关系；Decoder使用 N 个可学习目标查询并行解码结果；二分图匹配损失（匈牙利算法）将预测与真值最优匹配。

优势与局限

优势：简化流程，无需NMS；全局感受野，大目标检测效果好；易于扩展到开放词表检测。局限：训练收敛慢（需500 epoch）；小目标检测差；计算量大。DINO、Deformable DETR等改进缓解了上述问题。

从DETR到Grounding DINO

DETR证明了Transformer可以端到端解决检测问题，为后续开放词表检测（Grounding DINO）、指代检测（Referring Expression Comprehension）等任务奠定了基础。Grounding DINO将detection与grounding统一到一个框架，实现了"用自然语言描述检测目标"的能力。

二分图匹配损失原理

匈牙利算法计算预测集合与真值集合的最小匹配代价。匹配代价 = 分类负对数似然 + 边界框L1损失 + GIoU损失。该机制使模型无需NMS即可自动避免重复检测。

YOLO核心哲学与网格划分

核心哲学："只看一次"（You Only Look Once）。模型对输入图像执行单次前向传播，同时输出所有目标的边界框位置、类别和置信度。检测从"分类+定位"的组合问题简化为一个端到端的回归问题。与两阶段"先看哪里可能有目标，再看是什么"不同，YOLO一眼看完，同时回答位置和类别。

YOLOv1 网格预测方案

输入图像划分为 S × S 网格（S=7）。每个网格负责检测中心落在该网格内的目标。输出张量形状：S × S × (B×5 + C)。以PASCAL VOC（20类）为例：7×7×(2×5+20)=7×7×30。

现代YOLO的演变

YOLOv3及以后不再使用固定网格，而是在多尺度特征图每个位置密集预测。每个位置对应原图的一个区域，通过锚框覆盖不同形状和尺寸的目标。网格越密（特征图分辨率越高），可检测的目标越小。

核心理解

网格划分的本质是将空间位置离散化。每个网格/特征图位置相当于一个"检测锚点"，模型在该位置预测以该点为中心的目标。YOLO的关键突破在于证明了"一个统一的卷积网络可以同时预测边界框和类别概率"这一设计范式的可行性。

预测框数学表示与损失函数

一个完整的检测框预测包含三个组成部分，各自承担不同的语义角色：

bbox = (x, y, w, h) # 边界框坐标（中心点 + 宽高），相对于网格归一化 objectness = P(Object) # 该位置存在目标的概率，取值范围 [0,1] class_probs = P(Class|Object) # 给定目标条件下的类别分布，使用 Softmax/Sigmoid score = objectness × max(class_probs) # 最终置信度分数，用于 NMS 排序

Objectness 目标置信度

表示预测框包含目标的概率。训练时，若锚框与真值框IoU超过0.5设为1，否则设为0。推理时用于过滤低质量预测。Objectness使模型学会区分"有目标"和"只有背景"两种状态。

损失函数三部分构成

框回归损失：MSE或CIoU损失，权重最大。预测框与真值框的坐标差异。Objectness损失：二分类交叉熵。分类损失：交叉熵。三部分加权求和，通常框回归损失占比最高。

# YOLO 损失函数（简化版本） loss_box = λ_coord * Σ(2 - w_i * h_i) * CIoU_loss # 边界框回归 loss_obj = λ_obj * Σ BCEWithLogits(objectness, target_obj) # objectness loss_cls = λ_cls * Σ BCEWithLogits(class_probs, target_cls) # 分类 total_loss = loss_box + loss_obj + loss_cls

锚框（Anchor Box）原理与聚类

锚框是预设的候选框形状，作为边界框回归的参考基准。模型在锚框基础上预测偏移量而非绝对坐标，这降低了回归难度且提高了训练稳定性。若没有锚框，模型需从零学习预测任意形状的框，梯度更新方向不稳定。

为什么需要锚框

目标尺度差异大（如行人vs汽车相差数十倍），长宽比多样（横向车辆、纵向行人、方形标志）。锚框提供了先验形状约束，使模型专注于学习精确偏移。

K-means 聚类获取锚框

YOLO使用K-means从训练数据自动学习锚框尺寸。距离度量使用IoU：d(box, centroid) = 1 - IoU(box, centroid)。使用IoU距离确保聚类结果与检测性能直接相关。

# YOLOv5 默认锚框（COCO数据集聚类结果） anchors: - [10,13, 16,30, 33,23] # P3/8 小尺度特征图 → 小目标 (80×80) - [30,61, 62,45, 59,119] # P4/16 中尺度特征图 → 中目标 (40×40) - [116,90, 156,198, 373,326] # P5/32 大尺度特征图 → 大目标 (20×20) # 锚框重聚类命令（更换数据集后必做） python utils/autoanchor.py --data data/custom.yaml --cfg models/yolov5s.yaml
实践关键提示

更换数据集后务必重新聚类锚框。安全帽检测等专用数据集中目标普遍较小（占图像面积<5%），默认COCO锚框偏大，需聚类得到[4,5, 8,10, 12,15]等适合小目标的锚框。使用不匹配的锚框会导致mAP下降3-5个百分点。

Backbone：CSPDarknet53特征提取

Backbone是检测器的"眼睛"，负责从输入图像提取多层次视觉特征。YOLOv5使用CSPDarknet53，融合了Cross Stage Partial（CSP）结构和Darknet残差设计。CSP结构将特征图拆分为两部分，一部分经卷积处理，另一部分直接拼接，既减少计算量又保证梯度流通。

浅层特征（高分辨率）

靠近输入层的特征图分辨率高（160×160），含丰富的空间细节和边缘信息。感受野小，适合检测小目标但语义信息较弱。对应P2/P3输出。

深层特征（低分辨率）

靠近输出层的特征图分辨率低（20×20），含高级语义信息。感受野大，适合检测大目标和类别判断。对应P5输出。

# YOLOv5s Backbone 结构（640×640 输入） Focus(3, 32, k=6, s=2) # 640→320 切片下采样（减少信息丢失） Conv(32, 64, k=3, s=2) # 320→160 特征图 P1 CSPLayer(64, 64, n=1) # 160→160 残差块堆叠 P2 Conv(64, 128, k=3, s=2) # 160→80 P3 ← 小目标检测分支 CSPLayer(128, 128, n=3) # 80→80 增强特征表达 Conv(128, 256, k=3, s=2) # 80→40 P4 ← 中目标检测分支 CSPLayer(256, 256, n=3) # 40→40 Conv(256, 512, k=3, s=2) # 40→20 P5 ← 大目标检测分支 CSPLayer(512, 512, n=1) # 20→20 SPP(512, 512) # 空间金字塔池化（多尺度感受野）

Neck：FPN+PAN 双向特征融合

Neck位于Backbone和Head之间，负责融合不同尺度特征。YOLOv5使用FPN+PAN结构实现双向特征融合：FPN自顶向下传递语义信息，PAN自底向上传递定位信息。

P5 20×20 ↑上采样 P4 40×40 ↑上采样 P3 80×80 FPN: 深层语义 → 浅层
P3 80×80 ↓下采样 P4 40×40 ↓下采样 P5 20×20 PAN: 浅层定位 → 深层

FPN 自顶向下路径

将深层语义信息通过上采样传递到浅层（P5→P4→P3），增强浅层特征图"看懂内容"的能力。解决浅层特征"有细节但语义弱"的问题。每个融合层与对应Backbone层通过横向连接（1×1卷积）相加。

PAN 自底向上路径

将浅层定位信息通过下采样传递到深层（P3→P4→P5），增强深层特征图"找准位置"的能力。解决深层特征"语义强但定位模糊"的问题。两条路径形成完整的双向信息通道。

Head 检测头与完整流水线

Head将Neck融合后的特征映射为最终检测结果。YOLOv5使用解耦检测头，分别预测边界框回归、objectness和类别概率，避免分类和回归任务竞争特征表示。

CSPDarknet53 → Neck
FPN+PAN → Head×3
检测头 → 25,200候选

解耦头的设计优势

分类分支关注"目标是什么"，回归分支关注"目标在哪"。在YOLOv8中进一步改进为Task-Specific Decoupled Head，分类和回归分支使用独立的卷积层而非共享，各自学习最相关的特征表示，避免了任务冲突导致的梯度干扰。

多尺度检测原理与对应关系

目标检测必须处理不同大小的目标：一张图像中可能同时包含占据画面大半的车辆和仅数十像素的行人。YOLO在多个尺度的特征图上分别预测，每个尺度负责特定大小范围的目标。核心设计原则是"用合适的特征图检测合适大小的目标"。

特征图尺度下采样倍数每像素对应原图负责目标大小锚框范围

小目标检测的核心挑战

小目标在图像中仅占数十到数百像素。在20×20特征图上，一个32×32像素的小目标仅对应约1个特征点，信息量严重不足。因此小目标检测依赖高分辨率特征图（80×80或更高）。改进方向包括：高分辨率输入（如1280×1280）、特征增强模块（如注意力机制）、生成式数据增强、专门的小目标检测层。

NMS 非极大值抑制算法

模型对同一目标会输出多个高度重叠的候选框（相邻特征图位置、不同锚框都可能检测到同一目标）。NMS从这些重叠框中选出最优的一个，删除冗余框。这是YOLO推理流水线中的关键后处理步骤。

  1. 将所有检测框按置信度分数从高到低排序
  2. 选择分数最高的框，保留为最终检测结果
  3. 计算其余框与保留框的IoU（交并比）
  4. 删除IoU超过阈值（通常0.5）的框——它们检测的是同一目标
  5. 从剩余框中重复步骤2-4，直到没有候选框
def nms(boxes, scores, iou_threshold=0.5): """非极大值抑制 —— 完整实现""" # boxes: [N, 4] (x1,y1,x2,y2); scores: [N] indices = scores.argsort(descending=True) # 按置信度降序排列 keep = [] while len(indices) > 0: i = indices[0] # 取当前最高分框的索引 keep.append(i) # 计算其余框与保留框的 IoU ious = compute_iou(boxes[i], boxes[indices[1:]]) # 保留 IoU < 阈值（不重叠）的框 mask = ious < iou_threshold indices = indices[1:][mask] return keep # 保留框的索引列表

NMS 阈值影响与改进变体

NMS阈值是影响检测结果的关键超参数，在召回率和精确率间取得平衡。不同场景需要不同的阈值策略。

阈值过低（< 0.3）

相邻的不同目标可能被误删。密集人群中两个紧挨的行人IoU可达0.4，过低阈值导致其中一个被错误抑制。召回率下降，漏检增多。

阈值适中（0.5）

通用场景的默认选择。在COCO等通用数据集上经广泛验证，召回率和精确率平衡较好。YOLO系列默认值。

阈值过高（> 0.7）

同一目标保留过多重复框，假阳性增多。精确率下降，后处理计算量增大。适用于重叠容忍度高的场景。

改进变体对比

方法核心思想适用场景
Soft-NMS不直接删除重叠框，根据IoU衰减其置信度。IoU越大衰减越强，避免"一刀切"漏检密集人群、细胞检测等严重重叠场景

推理流程：图像预处理与Letterbox

预处理步骤直接影响检测精度。错误的预处理会导致目标变形、位置偏移或检测失败。YOLO使用Letterbox缩放保持原始宽高比。

  1. 读取图像：使用OpenCV或PIL读取，保持原始宽高比
  2. Letterbox缩放：等比例缩放到目标尺寸（640×640），不足部分用灰色(114,114,114)填充。保持宽高比不变，避免目标形变
  3. 像素归一化：像素值从[0,255]缩放到[0,1]（除以255.0）
  4. 维度调整：HWC → CHW，添加batch维度，最终形状[1, 3, 640, 640]

直接拉伸 vs Letterbox

将1280×720直接拉伸为640×640，宽高比从16:9变为1:1，圆形目标变为椭圆。Letterbox保持形状不变，填充灰色区域不会引入干扰信号。

坐标反映射

模型输出框在640×640坐标系中，需通过缩放比例scale和填充偏移(dw, dh)映射回原图坐标系。映射公式见右侧代码。

def preprocess(image_path, input_size=640): img = cv2.imread(image_path); img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB) h, w = img.shape[:2] scale = min(input_size / h, input_size / w) # 保持宽高比的缩放比 new_h, new_w = int(h * scale), int(w * scale) resized = cv2.resize(img, (new_w, new_h)) dw, dh = (input_size - new_w)//2, (input_size - new_h)//2 # 填充偏移 canvas = np.full((input_size, input_size, 3), 114, dtype=np.uint8) canvas[dh:dh+new_h, dw:dw+new_w] = resized canvas = canvas.astype(np.float32) / 255.0 # 归一化 canvas = canvas.transpose(2, 0, 1)[np.newaxis, ...] # CHW + Batch return canvas, (scale, dw, dh, h, w) # 返回元数据供后处理

前向传播与解码

预处理后的图像送入模型前向传播，得到原始输出张量。需经过解码将张量转换为可读的检测坐标。模型输出的 xy 是相对于网格的偏移（0~1），wh 是相对于锚框的缩放因子。

import torch model = torch.load('yolov5s-lite.pt', map_location='cpu')['model'].float() model.eval() with torch.no_grad(): outputs = model(torch.from_numpy(input_tensor)) # list of 3 tensors def decode_output(pred, anchors, stride): """将模型原始输出解码为边界框坐标""" bs, _, ny, nx = pred.shape # 如 [1, 255, 80, 80] pred = pred.view(bs, 3, 85, ny, nx).permute(0, 1, 3, 4, 2) pred[..., 0:2] = pred[..., 0:2].sigmoid() # xy: 网格内偏移→sigmoid归一化 pred[..., 4:] = pred[..., 4:].sigmoid() # objectness + 类别分数 grid_y, grid_x = torch.meshgrid(torch.arange(ny), torch.arange(nx)) xy = (pred[..., 0:2] + torch.stack([grid_x, grid_y], dim=-1)) * stride wh = pred[..., 2:4].exp() * anchors * stride # 锚框缩放 + 步长还原 return xy, wh, pred[..., 4:5], pred[..., 5:] # xy, wh, obj, cls_probs
解码要点

xy 加上网格坐标再乘以步长得到原图坐标；wh 乘以锚框尺寸和步长得到实际宽高；Objectness和类别分数需通过Sigmoid激活。注意网格坐标 grid_x, grid_y 的形状需与预测张量对齐。

NMS后处理与坐标映射

解码后的候选框约25,200个，需经置信度过滤 + NMS去重 + 坐标映射回原图。三个后处理步骤缺一不可。

def postprocess(predictions, conf_thres=0.25, iou_thres=0.5, scale, dw, dh): """完整后处理：合并→筛选→NMS→坐标映射""" boxes, scores, class_ids = [], [], [] for pred in predictions: # 遍历3个尺度 xy, wh, obj, cls_probs = pred conf = obj * cls_probs.max(dim=-1, keepdim=True)[0] # 综合置信度 mask = conf.squeeze(-1) > conf_thres if not mask.any(): continue xy, wh, conf, cls = xy[mask], wh[mask], conf[mask], cls_probs[mask].argmax(dim=-1) boxes.append(torch.cat([xy - wh/2, xy + wh/2], dim=-1)) # [x1,y1,x2,y2] scores.append(conf.squeeze(-1)) class_ids.append(cls) if not boxes: return [] boxes, scores, class_ids = torch.cat(boxes), torch.cat(scores), torch.cat(class_ids) keep = torch.ops.torchvision.nms(boxes, scores, iou_thres) # NMS results = [] for idx in keep: x1 = max(0, (boxes[idx,0]-dw)/scale); y1 = max(0, (boxes[idx,1]-dh)/scale) x2 = min(orig_w, (boxes[idx,2]-dw)/scale); y2 = min(orig_h, (boxes[idx,3]-dh)/scale) results.append({'box':[x1,y1,x2,y2], 'score':scores[idx].item(), 'class':int(class_ids[idx])}) return results
坐标映射关键公式

模型输出坐标 → 原图坐标：(box_coord - padding_offset) / scale。其中padding_offset为Letterbox填充的(dw, dh)，scale为缩放比例。裁剪到[0, orig_w]和[0, orig_h]防止越界。

检测评估指标：mAP / IoU / F1 Score

理解评估指标是正确解读模型性能的前提。目标检测使用多个互补指标全面衡量模型表现。

IoU（交并比）

预测框与真值框重叠程度的度量：交集面积/并集面积。用于判断检测是否正确（正样本判定依据）和NMS筛选。IoU≥0.5通常视为正确检测。

mAP（平均精度均值）

对所有类别计算Average Precision后取平均。mAP@0.5使用IoU=0.5阈值，mAP@0.5:0.95在[0.5,0.95]区间以0.05步长取10个阈值的平均值，后者是COCO官方标准，更严格。

F1 Score

精确率（Precision）和召回率（Recall）的调和平均。F1 = 2×P×R/(P+R)。反映模型在精确率和召回率间的平衡情况。F1最高对应的置信度阈值通常为最优阈值。

# 计算 mAP 的核心流程 def compute_ap( detections, ground_truths, iou_threshold=0.5): # 1. 按置信度降序排列所有检测结果 # 2. 对每个检测框，计算与同一类别真值框的 IoU # 3. 若 IoU ≥ 阈值→True Positive，否则→False Positive # 4. 计算累积 Precision 和 Recall 曲线下面积（AUC）= AP # 5. 对所有类别 AP 取平均 = mAP pass # 完整实现约 100 行，详见 pycocotools 库

YOLO数据增强策略

数据增强是YOLO训练管线的重要组成部分，对模型泛化能力影响显著。YOLOv5使用多种数据增强组合，以有限的标注数据产生多样化的训练样本。

增强方法操作方式作用
Mosaic将4张图像随机裁剪后拼接为1张，同时拼接对应的边界框标注大幅增加小目标样本数量，提升小目标检测能力
MixUp两张图像按比例混合（alpha混合），边界框取并集提升模型鲁棒性，减少过拟合
HSV扰动在HSV色彩空间随机调整色相(H±0.015)、饱和度(S±0.7)、明度(V±0.4)增强对光照变化的鲁棒性
随机缩放/平移以0.5概率随机缩放(0.5~1.5倍)和平移(±0.1倍图像尺寸)增强尺度不变性和位置鲁棒性
随机翻转50%概率水平翻转图像和对应的边界框相当于训练数据翻倍，减少过拟合
